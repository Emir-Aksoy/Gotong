This file records the people who build AipeHub — and **every kind of
contribution counts, not only code.** Writing the tutorial that finally makes
federation click, giving the talk that brings fifty people to the project,
translating the docs into a language the core team doesn't speak, running the
room where newcomers get unstuck — that is the work that turns *a good framework
nobody finds* into *a good framework people actually use*, and it is recognized
here with the same standing as a merged PR.

This is the people-facing half of [pillar ⑤ of the Recognition
System](docs/RECOGNITION-SYSTEM.md#pillar-5).
Its currency is the project's only currency: **honest, visible attribution** —
no money, no token, no bounty. And it is **a record, not a ranking.** It states
*what you did*, in the open, with your name on it. It does not sort people by a
number, because audience size is gameable and chasing it pulls a project toward
vanity — see [What we don't do](docs/RECOGNITION-SYSTEM.md#4-what-we-dont-do-honest-boundary).

## How a contribution is recorded

We use the [All Contributors](https://allcontributors.org) emoji vocabulary as a
shared, recognized language for contribution kinds — but **not** its bot or
GitHub Action. This repo runs no Actions budget on bookkeeping, and a
hand-maintained markdown table is the lightest honest thing that works. You are
added by a normal pull request.

| Emoji | Kind | Emoji | Kind |
|---|---|---|---|
| 💻 | Code | 📹 | Video |
| 📖 | Documentation | 📢 | Talk |
| 🌍 | Translation | ✅ | Tutorial |
| 🎨 | Design | 💬 | Answering questions / community support |
| 👀 | Review | 📋 | Event organizing |
| 🤔 | Ideas & planning | 📝 | Blog post / writing |
| 🐛 | Bug reports | 🚇 | Infrastructure |
| 🚧 | Maintenance | 🔌 | Adapter / connector |

The four reach kinds — 🌍 translation, 📹 video, 📢 talk, ✅ tutorial, 📝 blog,
💬 community, 📋 events — sit deliberately in the **same table** as 💻 code. The
point of pillar ⑤ is that they are not a lesser column.

## How to get added

1. **Open a pull request** adding a row — yours or someone else's. Crediting
   another person's work is welcome and encouraged.
2. **The bar is a real, honest contribution** of any kind in the legend above.
   For code/docs, the PR itself is the evidence. For a **reach** contribution
   (a video, talk, tutorial, post, translation, or sustained community help),
   **link the artifact** in the PR description so a maintainer can see it.
3. **A maintainer merges it** the same way any small PR merges (lazy consensus,
   see [`GOVERNANCE.md`](GOVERNANCE.md#how-a-change-lands)). No bot, no ceremony.

If your contribution is a piece of **learning material** — a video, tutorial, or
talk that teaches AipeHub well — it can *also* be curated into
[`LEARN.md`](LEARN.md), the showcase newcomers are pointed to. Being recorded
here and being curated there are two different acts of recognition; a great
tutorial earns both.

## Standing, not just a row

Sustained reach work is not only recorded here — it earns a real say in the
project. `GOVERNANCE.md`'s [maintainer
ladder](GOVERNANCE.md#becoming-a-maintainer) counts an **equivalent track** to
code: keeping the docs alive in several languages, running the community,
sustained educational material. A person who never writes a line of framework
code but does that work week after week is contributing exactly the kind of
sustained judgement the ladder is meant to recognize.
