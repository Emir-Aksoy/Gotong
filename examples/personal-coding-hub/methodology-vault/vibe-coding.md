# Vibe coding —— 凭感觉写(以及它的边界)

> 源:[[raw/sources]] · 上级:[[index]]

## 是什么

Karpathy 提出的词:**完全顺着感觉走** —— 把目标抛给 agent,接受它给的 diff,
**基本不读代码**,出错了就再抛个 prompt 让它修。是一种「忘掉代码存在」的写法。

## 什么时候用

- ✅ 周末项目、一次性脚本、原型、探索一个想法能不能成。
- ✅ 风险低、可丢弃、不用维护的东西。
- ✅ 你想**快**,且错了代价小。

## 什么时候**别**用

- ❌ 要长期维护、别人要读、上生产的代码。
- ❌ 安全/资金/数据相关(本 hub 用 `dangerousCommandGate()` 兜底,但那是底线不是
  许可证)。
- ❌ 你已经积累了一堆「自己都看不懂」的代码 —— 那是该切到 [[agentic-engineering]]
  的信号。

## 关键张力

vibe coding **快**,但会攒下**没被理解的代码**:能跑,直到不能跑,然后没人会修。
Karpathy 自己的态度是:**原型凭感觉,要留下的东西换挡**。

> 经验法则:**vibe-code 探针(spike),agentic-engineer 留存(keeper)**。
> 同一个目标,先凭感觉打个原型验证方向,确认要留 → 重做成小步、可审、带测试的版本。
