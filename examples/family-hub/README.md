# family-hub — 家庭 hub 开箱包（FAM-M1）

一家人共用 hub 的一键装模板。它是
[`docs/zh/FAMILY-HUB.md`](../../docs/zh/FAMILY-HUB.md)（家庭 hub 一页纸 +
「开箱 15 分钟」验收标准）的配套工件：把 15 分钟阶梯里 **10→15 分钟那一段
（第一张审批卡）**从「自己想怎么配」压成「装完照按钮点」。

## 装什么

| 件 | 说明 |
|---|---|
| 2 个托管 agent | 家庭帮手（整理申请 / 出执行方案）+ 家庭晨报员（诚实模式：没数据不编） |
| `family-approval-demo` 工作流 | 家人在 `/me` 发起「想让 AI 帮家里办件事」→ AI 整理一页申请 → **停在指定家长的真人审批步** → 批准才出方案，拒绝就不做，重启卡还在。这就是验收 checklist 的**时刻 C** |
| `family-brief` 工作流 | 家庭晨报，`/me` 一键跑；带每早 7 点定时建议（只带节奏不带人，装完「定时」卡补人启用） |
| 3 个生活连接器槽位 | 日历 / 笔记 / 任务，全部可选——不挂照样跑，晨报进诚实模式 |
| 1 条 golden-run 验收 | 装完面板「跑验收」零 LLM 判卷（审批流刻意不进验收：它要验的就是「家长真人批一次」，见模板头注释） |

## 怎么装

- **面板**：admin →「工作流 / 模板画廊」→「家庭 hub」→ 安装（导入时填一次 DeepSeek key）。
- **一条命令开荒**（含补人启用定时）：

```bash
gotong provision examples/family-hub/template/family-hub.template.yaml \
  --url http://127.0.0.1:8745 --token <admin-token> --user <成员id>
```

## 装完点什么（对照 15 分钟验收）

1. 孩子设备登录 `/me` →「请家里的 AI 办件事」→ 填想办的事 + 家长用户名 → 提交。
2. 家长的 `/me` 收件箱出现「家庭事项审批」卡（**时刻 C**：卡先出现，动作没发生）。
3. 家长批准 → 工作流续跑出执行方案；换一次点拒绝 → 流到此为止，什么也没做。
4. 「来份家庭晨报」一键跑通即 golden-run 同款路径；「定时」卡补人启用后每早自动跑。

真实的对外动作（发消息 / 花钱）不在演示流里——模板不带出站凭证、绝不预授权；
挂了连接器之后，那些动作仍逐次过审批闸（接入 ≠ 授权行动）。

深水区（孩子长大要自己的主权 hub）：见画廊里的 `family-tutor` + `child-desk`
双模板与 [`docs/zh/FAMILY-LEARNING-HUB-DESIGN.md`](../../docs/zh/FAMILY-LEARNING-HUB-DESIGN.md)。
