# Gotong 环境旋钮注册表 (GOTONG_*) — GUARD-M1 承重门的锚
#
# 这里登记了运行时源码 (packages/<pkg>/src) 里读取的每一个 GOTONG_* 环境旋钮。
# scripts/env-registry-gate.mjs 会断言「源码引用的 GOTONG_* 集合」== 「本表集合」:
#   - 代码里加了新旋钮却没登记 → 门变红 (这正是防旋钮无声膨胀的护栏)。
#   - 本表留了源码已删的旋钮     → 门变红 (保持诚实)。
# 所以下一个旋钮无法悄悄落地——必须自觉编辑本表。
#
# 作用域 = 生产源码 (packages/<pkg>/src 下的 .ts) 里**非注释**的引用。两条精度规则:
#   1. 只出现在注释里的名字不算旋钮 (2026-07-20 剔除 3 个: GOTONG_A2A_AGENTS 是
#      Route-B P1-M11b 已删的 env blob、GOTONG_WS_HARDENING_PROFILE 是从没读过的
#      设想、GOTONG_MASTER_KEY_ 是 GOTONG_MASTER_KEY_* 通配片段)。
#   2. 形似旋钮但根本不是环境变量的, 行尾打 `# not-a-knob:` 标签: 仍然钉住
#      (出现/消失照样红门), 但不计进对外报的旋钮数。
# 测试脚手架里的 GOTONG_* 不算旋钮，不登记。每项「是什么 / 默认」的说明是 GUARD-M2
# 惯例文档的活儿，不在本表。
#
# 当前登记数: 113 个真旋钮 + 4 个 not-a-knob 形似项 = 117 行
#   (沿革: …109 →M-EMB1 +3 embedder→112 →M-GRAPH +1 links→113 →M-RECON +1 reconcile→114
#    →LIB-M4 +1 librarian→115 →VOICE-M3 +4 TTS→119 →ASR-M3 +1 ASR→120
#    →2026-07-20 计数精度修正 −3 注释项、−4 形似项分开计 →113)
