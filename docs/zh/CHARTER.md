## 1. AipeHub 是什么

**AipeHub 是 AI 时代「人—智能体—机构」三者之间链接的自托管底座。** AI + Person + Hub。

它不是一个智能体，也不是又一个智能体框架。它是它们*底下*的那一层——一个注册表、
一条消息总线、一个任务路由器、一条受治理的联邦链路、一份只追加的 transcript。一张
LangGraph 图、一个 CrewAI crew、一个 CLI 编码智能体（Claude Code、Codex）、一个外部
A2A 智能体、一个人，全都通过同一个抽象接进同一个房间：`Participant`（参与者）。Hub
负责让信号流动、让边界生效。它自己从不跑模型，所以每一个决策都留在参与者手里。

这套心智模型小到一只手就握得住：

- **一个故意做笨的 Hub。** 它路由消息、派发任务、持久化 transcript、发出事件、执行
  治理闸门。它不拥有智能体循环，也不替谁做判断。
- **只有一种参与者。** 一个人是 `Participant`，一个智能体也正是 `Participant`。这里没有
  「请求人类输入的工具」；人和智能体共享同一套任务、transcript 和长任务原语。
- **状态就是文件。** 一个工作区就是磁盘上的一个目录（`.aipehub/`）。删掉它，房间就没了；
  拷贝它，你就把整个房间交给了别人；重启，什么都不会丢。

正是这套组合——而非某一个聪明的协议——才是 AipeHub *本身*。

## 2. 北极星——三条不可动摇

三句话定义了这个项目。它们不是功能，是身份。改动其中任何一条，得到的都不是一个更好
的 AipeHub——而是一个不再是 AipeHub 的东西。

1. **框架不跑 LLM。** Hub 只路由、派发、记录、设闸。推理和决策住在参与者里。正因如此，
   这套系统才能同时做到「笨、可审计、可信」。
2. **人和智能体是同一个 `Participant`。** human-in-the-loop 不是外挂上去的；人是头等的
   参与者，可以被派一个任务、把它挂起、再恢复它——和智能体用的是同一套机制。
3. **状态是磁盘文件。** 主权不是一个开关。你的房间、你的凭证、你的 transcript、你的历史，
   是一个你拥有、可以拷走的目录。重启是透明的，因为没有别的东西需要恢复。

这三条是项目里最难改的东西（§10）。其余一切都为它们服务，都可商量。

## 3. 它为什么存在——三层链接

AipeHub 为三层链接建一个能干活的底座，按一个用户多快会触及它们来排：

**第 1 层——一个人和他自己的 AI。**「我的 AI 桌面」。一个人的 hub、私人 workflow、永远
不离开本机的凭证。目标是五分钟跑起来、不写代码、AI 替你做真正的事。

**第 2 层——跨边界的人与智能体。** 跨组织协作：多用户、角色、邀请、点对点联邦。一个
workflow 可以伸到一条组织边界的另一头，但凭证、数据、计费各自留在家里。跨过边界的
东西，受一份显式的 per-link 信任契约约束；而那些有后果的跨越，要等一个人拍板。

**第 3 层——框架自身。** 清晰、稳定、可适配。Hub 故意保持笨；`Participant` 始终是那唯一
一个抽象；协议、凭证、配额全都有显式、可见的边界。框架自己的活，是在不背叛第 1、2 层
的前提下，跟上 AI 奔跑的速度。

AI、智能体、以及如今正在到来的多智能体框架，会以惊人的速度叠进日常生活。最大的那几家
模型厂商、各个通信平台、以及其他既得者，正以同样的速度去占住那块地、把它做成一种垄断
——一个其全部价值就在于「向所有依赖它的人榨取」的杠杆位。AipeHub 存在，是为了让一个
替代选项留在桌面上：把 AI、智能体、workflow 整合进同一个栈、并以开源放出来，从而钝化
那根杠杆——任何人都能拿到它、部署它、自己跑它，没有守门人、没有租子、不必征求许可。

它为这样的人而存在：想让 AI *做点真正要紧的事*——管家、帮家人、管钱、协调团队——又
不愿把钥匙交给一片自己控制不了的云去做。

## 4. 信任楔子——为什么你敢把要紧的事交给它

大多数 AI 工具只给两个选项：把一切交给一片你控制不了的云，或者你自己把它们全接起来。
AipeHub 是第三个选项——你敢把它对准你的家、你的家人、你的钱，因为那些边界是真的、而且
是你的。三个属性让这成真，它们也是这个项目的护城河：

- **受治理。** 可逆的动作直接做；不可逆的——锁门、花钱、把一个孩子的数据发过一条链路
  ——要等一个人在收件箱里确认，而 workflow 跳不过这道闸。危险动作和跨边界动作，按构造
  就是 fail-closed。
- **本地。** 凭证加密住在你自己的 `.aipehub/` 目录里。和另一个 hub 联邦，共享的是一个
  *能力*，绝不是你的 vault。每个 hub 都保有自己的凭证库、自己的用量/成本账本。
- **公开。** 每一次派发、每一个结果，都是一份你能读的只追加 transcript。因为框架从不跑
  模型，所以没有一个需要你凭信任去接受的隐藏判断。

而且，权柄是你的，不是我们的。AipeHub 提供机制——闸门、vault、transcript——但策略由跑
这个 hub 的人来设定、来持有，不由这个项目来设。哪些动作算「有后果」、一份白名单放行
什么、谁来回应收件箱：是运营者来定，也由运营者持有这个决定。在家庭学习 hub 里，是家长
——不是 AipeHub——来设那道闸、并在闸前确认。我们不在你的决策桌上替自己留一个厂商的座位；
框架把操纵杆交到你手里，然后让到一边。

这就是那个楔子：不是「比别人更能干」，而是*对那些你绝不会交给黑箱的东西，可以托付*。

## 5. 愿景——它要去哪

终局是一张**自由图，不是一棵树。** 不是一个让租户在里面租位子的中心平台，而是许多
主权 hub 点对点互联——每一个都归运营它的人或组织所有，谁都不拥有别人的信任。一个控制面
可以*观察*（以 opt-in、保护隐私、只含计数的摘要），但它从不*接管*。这样一张图无法被
围住：没有中心可俘获、没有房东可收租、也没有任何一方的许可是你「继续运转」所必需的。

那张图上的参与者，不只是人和软件智能体。**具身智能**——机器人、传感器、那些在物理世界
里动作的机器——正沿着同一条时间线到来，而 AipeHub 生来就是要把它纳进来的：只要这样一个
系统讲 **MCP**，它就作为又一个 `Participant` 接进来，落在同一道受治理的边界之后——被派
任务、受同样的闸约束、写进和其他一切相同的那份 transcript。框架天然适合把人、智能体、
具身系统一并管理在同一个屋檐下，因为它本来就把它们当作同一种东西。

在那张图之上，长出一个**受治理的、可复用组件的市场**——模板、适配器、知识库连接器——
做到让你能把一整套能干活的架构装进一个文件交给别人，而别人敢用它，因为治理姿态随它一起
走、而知识内容不走。溯源是诚实的：一个模板带的是结构和引用，绝不带另一个组织的数据或
人员。

这个生态的货币是**荣誉，不是租子**（§7）。让贡献者持续贡献的，是诚实的署名和一条通向
话语权的路——而不是一笔钱。自由图之所以保持去中心，恰恰是因为不需要一个中心方来结一本账。

## 6. 怎么用它

AipeHub 在最适合一个人的那个界面上迎接他，而同一个 Hub 坐在所有界面背后：

- **个人模式**——五分钟，不写代码。导入一个旗舰模板，或者从一张表单建一个智能体；host
  替你把它拉起来。凭证留在你的磁盘上。
- **团队模式**——一个房间、多种角色（管理员 / worker / 智能体）、邀请、资源级 RBAC、
  成员在 `/me` 自助。
- **联邦**——每个组织跑自己的 hub；一条 `HubLink` 把其中两个连起来，受一份 per-link 信任
  契约约束（能力白名单 · data-class 闸 · 配额 · 撤销 · 知识库白名单）。workflow 可以跨过
  这条链路；两边的主权都原封不动。

界面之所以是复数的，是有意为之：一个浏览器管理 UI、`/me` 成员桌面、IM 桥（今天有
Telegram、飞书、Slack、QQ；Discord 和 Matrix 在计划中）、一个交互式 CLI/REPL、给工具和
外部客户端用的 MCP、以及一个可安装的 PWA。生态里凡有现成的开放协议，AipeHub 就讲那个
协议——**MCP**（工具与数据，双向）、**A2A**（智能体对智能体，双向）、**ACP**（驱动一个
hold 住的编码智能体会话）——而它自己只拥有一个：**HubLink**，两个 hub 之间那条受治理的
联邦链路。

模板还能装下整个 hub：一个文件打包 N 个智能体、N 条 workflow、可寻址的知识库槽位、以及
一键的 key 设置——是结构和接线，绝不是知识本身，也绝不是你的人员或密钥。

→ 从 [`OVERVIEW.md`](OVERVIEW.md) 起步 · 经 [`HANDS-ON-HUBS.md`](HANDS-ON-HUBS.md) 跑一个
现成的 hub · 经 [`AGENT.md`](AGENT.md) 接一个智能体。

决定都在公开场合做出，由那些已经对设计线有了感觉的人来做。权柄阶梯、一个维护者必须

没有币、没有赏金层，因为一个经济层会搅浑整个项目所立足的「文件优先、去中心」信任模型。
这套制度的四根支柱——引用排行榜、维护者晋升路径、便捷化共享、共享范本——汇总在

## 8. 非目标——它拒绝变成什么

一份宪章靠「排除什么」来锚定，和靠「承诺什么」一样多。AipeHub 刻意**不是**：

- **一个模型托管方。** 它不会把一个 LLM 推理循环养进 Hub 里。把你自己的模型接在中立的
  `LlmProvider` 接口后面。
- **一个拥有你的信任的中心 SaaS。** 它不会变成那种「拿着你的钥匙、你的数据、你的计费」
  当作使用代价的平台。控制面只观察；它不接管保管权。
- **一棵层级组织树。** 联邦是一张主权对等体的自由图，不是嵌在房东里的租户。没有哪个
  hub 在结构上高于另一个。
- **一套钱 / 币 / 赏金经济。** 激励层是荣誉（§7）；引入货币会把项目正在去中心化的那份
  信任重新中心化。
- **一个在暗处行动的自治循环。** 它不会让一个智能体在没有人类闸门的情况下做不可逆或
  跨边界的动作。「主动」永远不等于「在要紧的事上无人盯着」。

当一个提议的功能要求打破其中之一时，那是一场宪章级别的对话（§10），不是一个 pull request。

## 9. 它如何开放

整个框架是 **MIT 许可**——可以嵌进闭源和 SaaS 产品、可以修改、可以再分发，只要保留许可
文件和版权行（[`LICENSE`](../../LICENSE)、[`LICENSE-FAQ.md`](LICENSE-FAQ.md)）。社区模板各自
带 CC0/MIT 溯源，全部兼容商用。

AipeHub 发明的那些格式，是作为规范开放的，不只是作为运行的代码。workflow 与智能体定义
语言（`aipehub.workflow/v1`，以及智能体、团队、模板 manifest）和 **HubLink**——两个 hub
之间的联邦协议——是我们自己的设计，而它们属于所有人：任何人都可以读它们、实现它们、在
它们之上造一个相竞争的运行时、并把它装进商业软件里发行，不收费、不必征求许可。格式是一门
共享的语言，不是财产；这些只是项目的规范，除了「我们是把它们写下来的人」之外，我们对它们
不主张任何权利。它们的形状列在 **附录 A**（HubLink）和 **附录 B**（定义格式）。

这里的开放不只是一纸许可；它是架构本身。你的工作区是一个你能读、能拷、能备份、能带着走的
目录。没有锁定要逃，因为打从一开始就没有保管权这回事。

## 10. 修订这份宪章

这是一份活的文档，但它不会被随意编辑。一个改动伸得越深，门槛越高：

- **北极星（§2）** 只能由维护者和 steward 在公开场合、以显式而审慎的共识来修订，并把
  理由记录在案。它几乎永远不该改。一旦改了，这个项目就已经变成了另一个项目，而这件事
  应该被大声说出来。
- **其余一切** 按仓库其余部分演化的方式演化：一个把*为什么*讲清楚的 pull request，对照
  北极星审过，按 [`GOVERNANCE.md`](../../GOVERNANCE.md) 的 lazy consensus 合并。

每一个被采纳的修订，都会让本文档顶部记录的版本号往上跳一格（这和软件的版本号是两回事）：
一次普通改动是小版本跳一格，大版本跳格留给对北极星的改动——那唯一一种「意味着项目已经
变成别的东西」的改动。版本号就是读者用来知道自己手里拿的是哪一部宪法的东西。

如果你拿不准自己想造的东西合不合适，这份宪章就是那份用来跟它辩论的文档。它就是为此而存在的。

## 附录 A — HubLink，联邦协议

*这些附录概述每种格式当前的形状，供定位之用。权威的、带版本的规范住在所引的来源里，
其演化比这份宪章快；当两者不一致时，所引的规范管 wire、宪章管原则（§9）。*

**HubLink** 是两个 hub 之间那条受治理的链路：一条对称的、点对点的 WebSocket 通道，任一方
都可以向另一方派一个任务、发布一条消息，或关闭这条链路。它是 `ws://` / `wss://` 上的
JSON 帧，由 `MESH_PROTOCOL_VERSION` 标版本，并刻意与「智能体↔Hub」的 wire 协议
（[`PROTOCOL.md`](PROTOCOL.md)）分开——一条对等链路不需要那套协议的准入闸门或智能体注册表。

那些帧，每一个都是一个自包含的 JSON 对象，带一个 `type` 判别字段：

| 帧 | 方向 | 含义 |
|---|---|---|
| `MESH_HELLO` / `MESH_HELLO_ACK` | 握手 | 双向对等体认证（per-link 凭证，fail-closed） |
| `MESH_TASK` / `MESH_RESULT` | 任一向 | 派一个任务 / 返回它的结果（按 task id 配对） |
| `MESH_MESSAGE` | 任一向 | 发完即忘的消息 |
| `MESH_PING` / `MESH_PONG` | 任一向 | 保活 |
| `MESH_GOODBYE` | 任一向 | 协作式关闭 |

跨过这条链路的东西，受一份 per-link **信任契约**约束——能力白名单、data-class 闸、配额、
撤销、知识库白名单——这样一个 workflow 就能伸到一条组织边界的另一头，而凭证、数据、计费
在两边都留在家里。

HubLink 是 AipeHub 自己的设计，作为一份属于所有人的开放规范放出：用任何语言或产品去实现
它，商用与否皆可，不收费、不必征求许可。参考实现住在 `packages/transport-ws/`。

## 附录 B — workflow 与智能体定义格式

这些 YAML 格式，是一个 hub 的结构被写下来、被治理、被分享的方式。全都是 AipeHub 自己的
设计，全都以和 HubLink 相同的条款放出：一份属于所有人的开放规范——在任何地方实现它们，
商用与否皆可，不收费、不必征求许可。

- **`aipehub.workflow/v1`**——一条声明式 workflow：一个 `trigger`、一列 `steps`（每一步
  派给一个*能力*，而不是一个具名智能体）、一张由 `$ref` 接起来的派发图，以及控制流糖
  `when:`（条件）、`parallel:`（扇出）、`human:`（一个 human-in-the-loop 步骤）。可选的
  `surface.me` 把它暴露在成员桌面上，`governance` 声明它的风险姿态。这份 YAML 是一条
  workflow 受治理、带版本的根——每一次运行都钉在一个不可变的修订上，所以它永不漂移。
- **`aipehub.agent/v1`** 和 **`aipehub.team/v1`**——一份智能体 manifest，一个文件里装一个
  智能体或一整个团队：provider、模型、系统 prompt、能力、MCP server、一份子智能体派发
  白名单、以及一个可选的心跳。
- **`aipehub.template/v1`**——一个模板把 N 个智能体、N 条 workflow、可寻址的知识库*槽位*、
  以及一键的 key 设置，打包进一个可分享的文件。它只带结构和引用——绝不带知识内容、你的
  人员，或你的密钥。

参考实现：`packages/workflow/`（workflow runner 和 schema），以及
`packages/web/src/manifest.ts` / `template-manifest.ts`（智能体、团队、模板 manifest）。
