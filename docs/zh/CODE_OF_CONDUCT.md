# 行为准则（Code of Conduct）

<!-- doc-version: 1.0 -->
> **文档版本 1.0** · 中文译本 · 最后更新 2026-06-27 · 权威源：[English](../../CODE_OF_CONDUCT.md)。如译文与英文版冲突，以英文版为准。

Gotong 是个小项目，希望一直是个让人愿意待着的好地方。本文件说明我们**期待什么**、**不容忍什么**，以及有人越线时**会发生什么**。它刻意用我们自己的话写成——之所以短，是因为规矩本来就短。

## 标准

对待这个项目里的人——issue、pull request、Discussions、聊天里的人——请用你希望一位资深工程师在他状态糟糕的一天里对待你的方式：有耐心、直接、且尊重。你完全可以在代码上激烈分歧，同时依然对人友善。

**我们期待你：**

- 尊重不同的观点、经验水平和母语。本项目有讲中文和讲英文的参与者；（任一方向的）蹩脚英语都不是人格缺陷。
- 批评观点，不攻击人。「这个方案有竞态条件」是审阅；「你显然不懂并发」不是。
- 假定对方出于善意，并以善意回应。多数摩擦是误会，不是恶意。
- 接受维护者可能会说「不行」或「现在不行」，并且明白一个被关闭的 PR 不是对你个人的否定。

**我们不容忍：**

- 任何形式的骚扰、恐吓或歧视——包括基于年龄、体型、残障、族裔、性别特征、性别认同或表达、经验水平、国籍、外貌、种族、宗教或性取向。
- 性化的语言或图像，或不受欢迎的性关注。
- 人身或政治攻击、持续的捣乱，或蓄意的恐吓。
- 未经本人明确许可而公开他人的私人信息。
- 任何其他在专业场合中被合理认为不当的行为。

本准则适用于所有项目空间，也适用于你在公开场合代表本项目的时候。

## 举报

如果你经历或目睹了违反本准则的行为，请**私下向维护者举报**。你没有义务先公开与对方对质。

> **临时举报渠道（1.0 之前）：** 本项目尚无专门的行为准则邮箱。在此之前，请使用 GitHub 仓库上的**私密举报**（Security → report）来私密地联系维护者，并把举报标注为行为准则问题。建立一个专门的、标识清晰的行为准则联系方式是一项被跟踪的上线前待办——一旦它存在，本节会写明它。

举报将被保密处理。未经举报者同意，我们不会向被举报者透露举报者的身份。

## 执行

维护者负责维护本准则，并会本着善意回应举报。视严重程度，回应可能是：

1. **私下提醒**——一次安静的纠正，说明哪里不对、为什么。
2. **警告**——一份正式通知，并写明若继续会有的后果。
3. **临时封禁**——限定时长地排除出项目空间。该期间内不得与涉事人员互动。
4. **永久封禁**——针对严重违规，或警告之后仍持续的模式。

不本着善意维护本准则的维护者，自身也可能面临后果，由其他维护者和监护人决定（见 [`GOVERNANCE.md`](GOVERNANCE.md)）。

我们会倾向于采用能解决问题的**最简单**回应。目的是一个健康的项目，不是一个审判庭。

## 适用范围与精神

本准则刻意保持轻量。如果出现了它字面上没覆盖到的情况，请按其**精神**来判断：一个有分寸的人会认为这是尊重且善意的吗？如果不会，那它就出界了——不管有没有被列出来。
