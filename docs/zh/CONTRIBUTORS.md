# 贡献者（Contributors）

<!-- doc-version: 1.0 -->
> **文档版本 1.0** · 中文译本 · 最后更新 2026-06-27 · 权威源：[English](../../CONTRIBUTORS.md)。如译文与英文版冲突，以英文版为准。

这份文件记录建设 AipeHub 的人 —— 而且**每一种贡献都算数，不只是代码。** 写出
那篇终于把联邦讲明白的教程、做那场把五十个人带进项目的演讲、把文档翻译成核心
团队不会说的语言、守着那个让新人不再卡壳的房间 —— 正是这份工作，把*一个没人
发现的好框架*变成*一个真正有人用的好框架*，它在这里得到与一个合并 PR **同等的
认可**。

这是[荣誉激励制度支柱
⑤](RECOGNITION-SYSTEM.md#pillar-5)面向人的那一半。它的货币就是项目唯一的货币：
**诚实、看得见的署名** —— 不发钱、不发币、不发赏金。而且它是**一份记录，不是
一个排名。** 它陈述*你做了什么*，摆在明处、署你的名。它不按某个数字给人排序，
因为受众规模可刷，追逐它会把项目往虚荣那边拽 —— 见
[不做什么](RECOGNITION-SYSTEM.md)。

## 一次贡献怎么被记录

我们把 [All Contributors](https://allcontributors.org) 的 emoji 词汇当作一套
共享、被公认的「贡献类型」语言来用 —— 但**不用**它的 bot 或 GitHub Action。
这个仓库不在记账上花 Actions 预算，一张手工维护的 markdown 表是最轻的、能成事的
诚实做法。你通过一个普通 pull request 被加进去。

| Emoji | 类型 | Emoji | 类型 |
|---|---|---|---|
| 💻 | 代码 | 📹 | 视频 |
| 📖 | 文档 | 📢 | 演讲 |
| 🌍 | 翻译 | ✅ | 教程 |
| 🎨 | 设计 | 💬 | 答疑 / 社区支持 |
| 👀 | Review | 📋 | 活动组织 |
| 🤔 | 想法与规划 | 📝 | 博客 / 写作 |
| 🐛 | Bug 报告 | 🚇 | 基础设施 |
| 🚧 | 维护 | 🔌 | 适配器 / 连接器 |

四类传播贡献 —— 🌍 翻译、📹 视频、📢 演讲、✅ 教程、📝 博客、💬 社区、
📋 活动 —— 刻意和 💻 代码放在**同一张表**里。支柱 ⑤ 的全部要点就是：它们不是
低人一等的一列。

## 怎么被加进来

1. **开一个 pull request** 加一行 —— 你自己的，或别人的。为别人的工作署名是
   受欢迎、被鼓励的。
2. **门槛是上面图例里任意一种真实、诚实的贡献。** 代码 / 文档的话，PR 本身就是
   证据。**传播**类贡献（视频、演讲、教程、文章、翻译，或持续的社区帮助），请
   在 PR 描述里**附上那件作品的链接**，好让维护者看得见。
3. **维护者合并它** —— 跟任何小 PR 一样合（lazy consensus，见
   [`GOVERNANCE.md`](../../GOVERNANCE.md#how-a-change-lands)）。没有 bot，没有
   仪式。

如果你的贡献是一份**学习材料** —— 一个把 AipeHub 教得好的视频、教程或演讲 ——
它*还*可以被策展进 [`LEARN.md`](LEARN.md)，那是新人被指过去的陈列。在这里被
记录、和在那里被策展，是两种不同的认可；一份出色的教程两样都该拿到。

## Standing，不只是一行记录

持续的传播工作不只在这里被记录 —— 它换来项目里真实的话语权。`GOVERNANCE.md` 的
[维护者晋升路径](../../GOVERNANCE.md#becoming-a-maintainer)把它算作与代码
**等价的一条路**：把文档在几种语言里持续养活、运营社区、持续的教育材料。一个
从不写一行框架代码、却周复一周做这件事的人，贡献的正是这套晋升路径本就想认可的
那种持续判断力。
