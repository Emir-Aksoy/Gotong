## 一、三十秒版本

家里一台小电脑（或云上一条命令），跑一个**全家共用的 AI 管家**（默认名字叫
「阿同」）。每个家人有**自己的账号**、自己的聊天入口——微信、Telegram 或网页——
和**自己的权限**。孩子可以随便问功课，但 AI 想**花钱、对外发消息、动重要东西**
的时候，会先在家长的收件箱里弹一张**审批卡**：批了才做，拒了不做，中途断电
重启也不会丢。所有密码和订阅 key 锁在一个**加密金库**里，家人之间互相看不见。

一句话：**AI 干活，家长掌闸，各人各账号，钥匙进保险箱。**

---

## 二、三个痛点，三个机制

下面三条就是 #8081 的原话，逐条对到 Gotong **已经做实**的机制——不是路线图。

### 1. 「给家人不同权限」→ 成员账号 + 角色

每个家人是 hub 里一个真实成员，角色四级：`owner`（家长本人）/ `admin` /
`member` / `viewer`。孩子拿 `member`：有自己的 `/me` 工作台、自己的任务收件箱、
自己跟管家的对话记忆；但管理面（加成员、配 AI、看金库、改闸门）在 `admin`
门后，孩子的界面里**根本没有那些入口**。权限不是「设置里一个开关」，是服务端
每个请求都校验的门（`requireAdmin`）。

### 2. 「凭证互不可见」→ 加密金库（vault）

AI 订阅 key、各种服务的 token，全部进 hub 的金库——信封加密落盘，界面和日志里
只出现 `${名字}` 占位符，明文不落配置文件（这条有防腐测试钉死）。孩子**用得上**
家长订阅的 AI（管家替他调用），但**拿不到** key 本身；家人之间的绑定凭证也互相
不可见。整个 hub 的状态就是磁盘上一个目录，搬家 = 复制目录，金库跟着走。

### 3. 「对外动作要有人批」→ 审批闸（governed 闸）

管家的工具分两类：**benign**（查资料、读日程、记笔记——直接做）和 **governed**
（对外发消息、花钱、改外部数据——**先停靠**）。governed 动作会变成一张审批卡，
落到指定家长的 `/me` 收件箱（也能推到家长的微信/Telegram）；批准后从停的那一步
继续，拒绝就不执行。停靠的任务写在磁盘上，**重启幸存、可以换人批**。这不是
「AI 记得要问一下」，是结构上够不着——闸在服务端，AI 绕不过去。

---

## 三、为什么流行方案做不到这个

不是它们不想，是架构不同：主流自托管管家是**单用户设计**（一个人、全权、明文
配置），多用户 RBAC 是开放 issue；云端助手有账号体系但**闸不在你手里**。Gotong
的内核里「人和 AI 是同一种参与者」，多成员、审批收件箱、金库、配额从 v4 起就是
承重墙——家庭场景不是加装，是本来的形状。

## 四、「开箱 15 分钟」验收标准

这是家庭版的 TTFR（time-to-first-result）尺子，对照仓库的 FUN 承重门
（`QUICKSTART.md` 五分钟漏斗 + `scripts/first-result-smoke.mjs`）设计：
**每一段有预算，每个承诺是一条独立可见的时刻**，任何一条不亮就算不通过。

**前置**：一台能跑 Docker 的电脑（或 Node ≥ 20），一部家长手机，一部孩子设备。
**全程不写代码、不手改配置文件。**

### 4.1 阶梯与预算

| 时段 | 做什么 | 必须看见什么（可见时刻） |
|---|---|---|
| **0 → 5 分钟** | `docker compose up`（或 `npx gotong start`）→ 浏览器自动打开设置向导 → 家长完成 owner 建号 + 粘一个 AI key | 面板首页亮起；管家「阿同」可以对话 |
| **5 → 10 分钟** | 面板里加第二个成员（孩子）→ 孩子用自己设备登录 `/me` | **时刻 A（权限分离）**：孩子的 `/me` 只有自己的任务与对话，没有任何管理入口；**时刻 B（凭证不可见）**：孩子界面不存在能读出 key 明文的地方 |
| **10 → 15 分钟** | 孩子让管家做一件「对外」的事（例：给某人发条消息）→ 家长收件箱弹审批卡 → 家长批准 | **时刻 C（对外有人批）**：卡先出现、批后才执行；再试一次点「拒绝」，动作确实没发生 |

### 4.2 试点家庭 checklist

给真实试点家庭（FAM-M2）照跑打钩用；A1–A7 全钩 = 通过。

- [ ] **A1** 起 hub 是一条命令，浏览器向导 5 分钟内完成，家长成为 owner
- [ ] **A2** 加第二个成员在面板里 ≤3 步完成
- [ ] **A3** 孩子设备登录 `/me` 后，看不到管理面/金库入口（时刻 A）
- [ ] **A4** 孩子端界面与日志里找不到任何 key 明文（时刻 B）
- [ ] **A5** 管家的对外动作先变成家长收件箱里的审批卡，而不是直接执行（时刻 C）
- [ ] **A6** 批准后动作续跑完成；拒绝则确实不执行；中途重启 hub 卡还在
- [ ] **A7** 全程 ≤15 分钟，零代码、零手改配置文件
- [ ] **A8**（加分）家人在微信/Telegram 里直接跟管家说话（IM 绑定成功）

### 4.3 尺子从哪来

FUN 门的形状是：**spawn 文档教的那条确切命令、剥掉一切作弊路径、逐条断言文档
许下的语义承诺、限预算、exit 0**。家庭版继承同样的纪律：A1–A7 每条对应叙事页
第二节的一个承诺；将来机械化成脚本时（FAM-M2 之后），断言的就是这些时刻。
