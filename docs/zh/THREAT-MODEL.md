# 威胁模型 — 把凭证和家人数据托付给 hub 之前，读这一页

> **这页给部署者看**（尤其是给家里装 hub 的家长）：你托付了什么、谁能看到什么、
> 结构上防住了什么、**防不住什么**、以及你必须做的三件事。

## 一、你托付了什么（资产清单）

装一个家庭 hub，进到这台机器/这个目录（`.gotong/`）里的东西：

| 资产 | 放在哪 | 形态 |
|---|---|---|
| AI 订阅 key（DeepSeek / Anthropic…） | 金库 `secrets.enc.json` | AES-256-GCM 信封加密，主钥单独放（`0600` 文件或环境变量） |
| 连接器凭证（日历 OAuth 令牌、笔记 token、IM bot token） | 金库（同上）；配置里只有 `${名字}` 占位符 | 同上；明文不落配置文件（有防腐测试钉死） |
| 家人和管家的对话、记忆、任务记录 | 每成员的记忆/任务文件 + transcript | 明文文件（**本机磁盘**，不上云——见 §二） |
| 审批记录（谁批了什么） | transcript + 停靠任务文件 | 追加式，运行时没有删除/改写 API |
| 成员账号（口令散列、角色、IM 绑定） | `identity.sqlite` | 口令散列存储；token 存 SHA-256 散列 |

**file-first 的另一面**：整个 hub 状态就是一个目录。复制目录=搬家（这是特性），
但也意味着**拿到目录+主钥的人拿到一切**（这是本页存在的原因）。

## 二、谁能看到什么（信任边界）

- **成员（如孩子）**：自己的 `/me` 工作台、自己的对话与任务。管理面（金库、成员
  管理、闸门配置）在 `requireAdmin` 服务端门后——成员界面**没有**这些入口，请求
  也过不去。成员看不到彼此的对话。
- **admin / owner（家长）**：能管一切配置，能看审批卡和运行记录。**这是信任模型
  的锚点：admin 是「家里管钥匙的人」，框架不防 admin**（SECURITY.md 明说
  「untrusted admins」超出范围）。给谁 admin，等于给谁钥匙串。
- **LLM 厂商**：管家/agent 的对话内容会发给你配的模型厂商（这是「用 AI」本身的
  含义）。选哪家厂商=选把对话给谁。
- **连接器厂商**：挂了云端连接器（Notion / Todoist / Google 日历…），对应数据会
  离开本机——面板对这类连接器**无条件显示「数据离开本机」披露**（`dataLeavesBox`
  原语），装之前看得见。
- **对端 hub（联邦）**：默认零联邦。建了边也只有**你策展过的能力**可被调用
  （能力白名单 + 数据类闸 + 出站审批闸），名片/发现永不自动建边，可选 PIN 公钥
  锚定对端身份。
- **腾讯/微信（若接了微信桥）**：iLink 条款明说腾讯仅收发不存储内容，但**收发
  经过它**——IM 桥的消息对平台方可见，这是所有 IM 桥（Telegram/飞书/微信…）的
  共性，不是 Gotong 能改变的。

## 三、结构上防住了什么（机制对照）

每条都是已落地机制，不是路线图；细节见 SECURITY.md「In-place mitigations」。

| 威胁 | 防它的结构 |
|---|---|
| 孩子/成员翻出订阅 key | 金库信封加密 + 界面只有 `${占位符}` + 读取面在 admin 门后 |
| AI 自作主张花钱/对外发消息 | governed 审批闸：动作先停靠成审批卡，批准才继续，拒绝不执行；停靠写盘**重启幸存**；框架本身不跑 LLM，AI 在结构上够不着「直接执行」 |
| AI 烧穿订阅额度 | 配额 fail-closed（额度耗尽=停，不是「继续跑再说」） |
| 冒充成员/偷 cookie | token 散列存储 + 常时比较；HttpOnly cookie；CSRF 双重校验（`GOTONG_ALLOWED_HOSTS`）；admin 口令暴力尝试限速 |
| 偷偷改历史（谁批过什么） | transcript 追加式，运行时无删改 API |
| 对端 hub 越权 | 能力白名单（未策展=拒）+ 数据类 allowlist（fail-closed）+ 出站需 owner 批 + 名片签名/PIN 验证（opt-in） |
| 恶意/买通的社区模板 | 模板导入走真解析器校验；画廊是显式白名单（CURATED）；模板带凭证占位而非明文（防腐测试） |

## 四、防不住什么（诚实节）

托付之前，这几条必须知道：

1. **拿到这台机器的人**。物理访问 / root / 文件系统权限在威胁模型之外——磁盘上
   的记忆、transcript 是明文，金库主钥也在盘上（除非你用环境变量供钥并另行保管）。
   **对策**：整盘加密（macOS FileVault / Linux LUKS）+ 这台机器别装来路不明的软件。
2. **恶意 admin**。框架不防管钥匙的人。**对策**：家里谁当 admin 想清楚；孩子是
   member 不是 admin。
3. **LLM 厂商侧**。对话发给厂商之后的事（留存/训练政策）由厂商条款决定，框架管
   不到。**对策**：选厂商=选条款；敏感话题可换本地模型（Ollama 已支持）。
4. **供应链**。npm 包**目前没有 provenance 构建来源证明**——我们是本机手动发布
   （结构性拿不到 provenance，启用路径已调研，见
   [`PUBLISH-RUNBOOK.md`](PUBLISH-RUNBOOK.md) 的 provenance 节）。第三方安全审计
   也还没有做过（内部审计纪律见 `audits/`）。**对策**：介意的话从 GitHub 源码构建
   而不是装 npm 包；关注版本锁定。
5. **认证用户的蓄意滥用**（应用层 DDoS、超大 payload）。限速是防误伤不是防内鬼。

## 五、家庭必做三件事

1. **备份**：定期跑 `gotong backup`（见 §六），备份文件放到**另一台设备/网盘**。
2. **两个 admin**：只有一个 admin 账号 = 手机丢了/口令忘了就锁死。给配偶或自己的
   第二设备再开一个。
3. **生产上锁**：出家门（公网可达）就把 SECURITY.md 结尾的 operator checklist
   过一遍——`GOTONG_COOKIE_SECURE=1`、`GOTONG_ALLOWED_HOSTS`、TLS 前置、
   admission gating 保持默认 `admin-approval`。

## 六、备份与恢复演练（搬家/换机/灾难恢复）

命令是 TS 原生的，Windows / 便携包同样能跑。**日常备份**（主钥不进档案——
档案被偷也解不开金库）：

```bash
gotong backup .gotong ~/backups
```

**搬家模式**（档案含主钥，**档案本身就是凭证**，传输/存放按密钥对待）：

```bash
gotong backup .gotong ~/move --include-master-key
```

**恢复**（先整档校验 sha256 清单，任何不符拒绝落盘、目标原样不动）：

```bash
gotong restore ~/backups/<档案>.tar.gz --space /新机器/gotong-space
```

**演练**（半年一次，10 分钟）：在另一台机器/另一个目录恢复最近一次备份 →
起 host → 登录 `/me` 看对话记忆还在 → 审批收件箱还在。**没演练过的备份等于
没有备份。** 日常备份恢复后金库需要你重新供主钥（这正是「主钥不进档案」的含义）。
