# 维护者（Maintainers）

<!-- doc-version: 1.0 -->
> **文档版本 1.0** · 中文译本 · 最后更新 2026-06-27 · 权威源：[English](../../MAINTAINERS.md)。如译文与英文版冲突，以英文版为准。

本文件列出有权审阅并合并 PR、分诊 issue、发布版本的人。各角色的含义以及维护者晋升阶梯，见 [`GOVERNANCE.md`](GOVERNANCE.md)——尤其是[成为维护者](GOVERNANCE.md#成为维护者)。

本文件刻意写得很短。AipeHub 是个年轻项目，目前只有一位维护者。这份文件存在的意义在于：让*第二位*维护者通过一条**写下来的路径**加入，而不是临时拍肩膀招呼一声——从而责任的归属永远不会变成一条不成文的习惯。

## 监护人（Steward）

监护人是有争议决策的最终仲裁者，也是[设计底线](GOVERNANCE.md#唯一不可妥协的底线)的守护者。

## 维护者（Maintainers）

维护者在各自负责范围内审阅与合并，并守护设计底线。当加入第二位维护者时（见[成为维护者](GOVERNANCE.md#成为维护者)），会在授予该角色的同一个 PR 里把他列入此处。

_目前除监护人外暂无他人。_

## 荣誉退任（Emeritus）

曾经的维护者，已退居二线，致以谢意。退任是一件**正常且光荣**的事——不是失败。

_目前暂无。_
