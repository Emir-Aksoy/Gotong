# 一人公司 hub — 一个人顶一个团队

> **这是一人公司垂直（OPC track）的首屏叙事页**：给独立开发者 / 自由职业者 /
> 独立顾问 / 一人工作室主理人读的一页说明 + 开箱验收标准。它回答两个问题：
> 这东西给我这个「一个人的公司」解决什么、多快能看到。深入设计与上线步骤见文末指针。
>
> 场景出处：这不是编的需求。「一人公司（one-person company / solopreneur）」是一个
> 真实且在扩张的人群——一个人靠工具杠杆做出过去一个团队的产出。他们的瓶颈从来不是
> 缺人手做具体的活，而是**一个人的注意力要同时劈给接客、交付、开票、理优先级、
> 维护自己的知识库**。AI 时代这正好是可以被「一个管家 + 几个专业分身」补上的缺口。
>
> Last updated: 2026-07-19（OPC-M1）

---

## 一、三十秒版本

一台小电脑（或云上一条命令），跑一个**你自己的 hub**：一个贴身管家（默认名字叫
「阿同」）+ 三个专业分身——**客户沟通助手、经营简报员、研究助手**。每天开工，
经营简报员先帮你理清今天的优先级、待跟进、现金流自查；来了新单，研究助手查你的
知识库出一页「这单怎么接、大概怎么报价」；要对外发点什么（客户邮件、报价、交付
说明），客户沟通助手拟成草稿、**停在你自己的审批步**，你过一眼签发才出最终文案，
不批就不发。所有 key 锁在**加密金库**里，对外动作**你签发**，产出全落在你自己的
磁盘上。

一句话：**AI 起草、你签发；分身干活、你掌闸；一个人养得起一个团队的调用量。**

## 二、三件一个人扛不匀的活，三个分身

一人公司的痛不是「不会做」，是「一个人分身乏术」。下面三条对到本模板**装完即用**
的三个分身——不是路线图，是画廊里点一下就装好的 [`examples/solo-company-hub`](../../examples/solo-company-hub)。

### 1. 每天开工先被杂事淹 → 经营简报员

一个人没有晨会、没有人帮你把昨天的尾巴和今天的重点摆出来。经营简报员每次被派到
就写一份简短经营简报，四个固定小节：**今日优先级 / 待跟进 / 现金流提醒 / 今日一问**。
配好定时后每早 8 点自动跑、跑完管家播到你的 IM。挂了日历 / 任务 / 客户资料的连接器
就读真实数据；没挂就进**诚实模式**——没有的数据绝不编造（不会伪造「今天的会议」
或「某客户欠款 X 元」）。

### 2. 来了新单不知道怎么接、怎么报价 → 研究助手

接单决策是一人公司最高频也最容易拍脑袋的地方。把新单需求贴进去，研究助手出一页
「这单怎么接」：**这单的背景 / 关键要点 / 报价参考 / 风险与注意**。挂了知识库连接器，
「报价参考」就用**你自己的历史项目和报价档案**做锚，而不是凭空一个数；没挂就按常识
给区间并说明依据。它查的是你知识库里的历史，措辞不冒充「当前市价」，也不替你做
接单决定——只把判断需要的信息摆全。

### 3. 对外沟通既要快又怕出错 → 客户沟通助手 + 你的审批步

一个人对外发的每封邮件、每张报价都代表公司，发错了没有同事帮你兜。
`client-outreach-approval` 工作流把这条路做成**结构化**的：你说想对外发什么 → 客户
沟通助手拟成一页草稿（发给谁 / 目的 / 拟发内容 / 需要留意）→ **停在你自己的审批步** →
你在 `/me` 收件箱过一眼、签发才出最终可发文案，拒绝就到此为止。一人公司老板**既是
发起人也是把关人**——AI 起草、你签发。

## 三、为什么这个形态适合一人公司

### 3.1 框架不跑 LLM，参与者才跑 → 便宜到一个人养得起

Gotong 的骨架只负责路由消息、派任务、写记录、管审批闸，**它自己不调 LLM**——每一次
模型调用都发生在参与者（你的分身）身上。这条设计立场对一人公司恰好是省钱的：三个
分身都跑 **DeepSeek**（`openai-compatible` / `deepseek-chat`），性价比甜区，一个人的
预算也扛得住一个小团队的调用量。要换更强或更省的模型、配主备降级链，都是面板里的
配置，不动代码。

### 3.2 对外动作在闸后 → 一个人也不会「AI 替我闯祸」

管家的工具分两类：**benign**（查资料、读日程、写自己的知识库——直接做）和 **governed**
（对外发消息、花钱、改外部数据——**先停靠等你批**）。这不是「AI 记得问一下」，是结构上
够不着：闸在服务端，AI 绕不过去。一个人没有风控同事，这道闸就是你的风控。

### 3.3 凭证进金库、状态是磁盘目录 → 你完全掌握

AI 订阅 key、各连接器 token 全部进加密金库，界面和日志里只出现 `${名字}` 占位符，
明文不落配置文件（有防腐测试钉死）。整个 hub 的状态就是磁盘上一个 `.gotong/` 目录，
搬家 = 复制目录，没有中央节点、没有厂商锁定。一人公司最怕的就是把身家绑在别人的
云上——这里的钥匙和数据都在你手里。

## 四、「开箱见产出」验收标准

对照仓库的 FUN 承重门（`QUICKSTART.md` 五分钟漏斗 + `scripts/first-result-smoke.mjs`）
设计：**每个承诺是一条独立可见的时刻**，任何一条不亮就算不通过。

**前置**：一台能跑 Docker 的电脑（或 Node ≥ 20），一个 DeepSeek key。**全程不写代码、
不手改配置文件。**

| 时段 | 做什么 | 必须看见什么（可见时刻） |
|---|---|---|
| **0 → 5 分钟** | `docker compose up`（或 `npx gotong start`）→ 向导建 owner + 粘 DeepSeek key | 面板首页亮起；管家「阿同」可以对话 |
| **5 → 8 分钟** | 画廊装「一人公司 hub」→ `/me`「来份今日经营简报」一键跑 | **时刻 A（分身能干活）**：四小节经营简报当场生成，无连接器也进诚实模式不报错 |
| **8 → 12 分钟** | `/me`「研究一个新单」贴需求 → 出一页「怎么接」 | **时刻 B（研究成锚）**：背景 / 要点 / 报价参考 / 风险四节齐全 |
| **12 → 15 分钟** | `/me`「起草一条对外沟通」→ 你自己的收件箱弹审批卡（卡上内联草稿全文）→ 签发 | **时刻 C（对外你签发）**：卡先出现、签发后才出最终文案；再试一次点「拒绝」，`finalize` 步被跳过、确实没出文案 |

前两条时刻（经营简报 / 新单研究）就是模板 `acceptance` 里的两条 golden-run——装完
面板点「跑验收」零 LLM 机器判卷，开箱不挂任何连接器也必须绿。时刻 C 刻意不进机器
验收：带 `human:` 步的流无人值守必然停在审批步，「你真签发一次」恰恰是它要人来验的。

## 五、往上兼容：一人公司 → 小团队

一个人做大了要招人、要接企业级数据源——这套形态是**往上兼容**的，不用推倒重来：

- **加人分权**：同一个 hub 加成员，角色四级 `owner`/`admin`/`member`/`viewer`，管理面
  在 `admin` 门后（服务端每请求校验，不是设置里一个开关）。招的第一个助理拿 `member`，
  有自己的 `/me` 和收件箱，看不到你的金库。
- **换企业级数据源**：知识库槽位从个人笔记换成企业级后端（`elasticsearch-kb` 走
  [`RAG-VIA-MCP.md`](RAG-VIA-MCP.md)），客户资料槽位接结构化数据库类 MCP server——
  槽位 id 不变，换的是后端。
- **多角色协作工作流**：画廊里的 `cafe-ops`（[`examples/cafe-ops`](../../examples/cafe-ops)）
  是多角色协作的参考形态——多个 agent 各司一职、跨角色派活。

> 这条「往上兼容」现在有了**完整企业级样板**：[`pro-firm-hub`](PRO-FIRM-HUB.md)
> （[`examples/pro-firm-hub`](../../examples/pro-firm-hub)）——多人分角色 + 三个分身
> 接力 + 企业级客户库 / 交付知识库全走只读 MCP + 负责人跨角色签发。它证明的正是同一个
> 内核从「一个人的公司」平滑长成「一个小团队」。但推广一人公司时**不必等它**：OPC
> 样板本身开箱即用，pro-firm-hub 是给「已经做起来、要招人接企业数据」的下一步准备的。

## 六、诚实盘点

| 维度 | 真实状态 |
|---|---|
| 功能成熟度 | ✅ 骨架承重墙齐备（多成员 / 审批闸 / 金库 / 定时 / 连接器 / 记忆），OPC 样板复用既有机制零内核改动 |
| 对外动作 | ⚠️ 模板不携带出站凭证、不预授权：批准后产出是**文案 + 记录**，不真的发送 / 动钱；要真发送需另挂连接器且**仍逐次过审批闸**（接入 ≠ 授权行动） |
| 报价参考 | ⚠️ 基于你知识库里的历史，**不是市场实时行情** |
| 部署摩擦 | ⚠️ 需一次 `docker compose up` / `npx gotong start` + 粘 key；面向愿意配置的 OPC 人群，非零门槛消费级 |
| 真人实测 | ❌ 尚无外部一人公司真机实测——推广即是找首批用户照第四节验收标准跑、逐条清摩擦 |

一人公司人群本身就有配置容忍度（他们习惯自己搭工具），这让 OPC 成为**冷启动的好
滩头**——上手门槛这个短板恰好被目标人群的特性抵消。
