**维度释义**：跨组织联邦＝不同主权 hub/组织间 agent 协作（凭证各归各家）；个人↔组织连续＝同一套从
个人模式平滑升到团队再到跨组织；框架不跑 LLM＝框架只路由/记账、不替参与者决策（Hub is dumb）。

**怎么读这张表**：
- **最右三列**（跨组织联邦 / 个人↔组织 / 框架不跑 LLM）**只有 AipeHub 是 ✅**，其余全 ❌/⚠️
  ——这是 AipeHub 真正的空位。
- 商业平台（上 4 行）：治理/HITL 满分，但**开源·自托管·数据自持全 ❌**（SaaS，厂商持有信任模型；
  部分提供 VPC 但本质仍是租户）。
- 开源框架/自托管平台（中 7 行）：开源·自托管满分，但**治理只 ⚠️、跨组织全 ❌**（单组织内）。
- 个人 agent（OpenClaw/Hermes/Goose/Odysseus）：自托管满分，但**治理·跨组织·连续路径全 ❌**（单人单机）。
- **AipeHub 是唯一一行全绿**——但这恰恰因为维度是按它的立场挑的，下面那张劣势表必须配着读。

## 二、劣势矩阵：诚实反向对比（换企业采购关心的维度，AipeHub 反过来弱）

上面那张表占优是「主场优势」。换一组企业采购真正会问的维度，差距立刻反向：

| 维度 | 谁强（具体产品） | AipeHub |
|---|---|---|
| 客户验证/规模 | Salesforce Agentforce(8000+ 客户)、ServiceNow | ❌ 早期/自用 |
| 生态集成(CRM/ITSM/Office/SAP) | Salesforce、ServiceNow、Microsoft | ❌ 靠 MCP 自己接 |
| 合规认证(SOC2/ISO/HIPAA) | 全部商业平台 | ❌ 无认证 |
| 开箱即用/无代码成熟度 | n8n、Dify、Agentforce | ⚠️ 要配置/example-first |
| 自带强模型 + SLA + 商业支持 | 全部商业平台 | ❌（且 by design 不跑模型，依赖你接的 MiMo/DeepSeek/Claude） |
| 可视化编排成熟度 | n8n、Flowise、Dify | ⚠️ 声明式 YAML 为主（已补只读 DAG 图） |

**网络效应短板（单列，因为它对联邦类产品是生死线）**：联邦的价值随对端数量超线性增长，
而冷启动时对端数＝0。这是所有「跨组织」产品的死亡陷阱——下面 §四会讲为什么我们选的目标用户
**自带对端、能绕过这个陷阱**。

## 三、一句话结论

**AipeHub 不是「更好的 Agentforce」或「更强的 n8n」，它在一个别人没占的交叉格子里**：
自托管主权 + 跨组织联邦 + 组织级治理/HITL + 个人到组织连续路径 + 框架不跑 LLM。代价是它在
「成熟度/生态/合规/客户验证」上是早期产品——这恰好是商业平台最强的地方。

> 数据来源：vdf.ai 厂商全景 / Futurum 平台战 / Zylos 协议层 / Knowlee 开源自托管 / Strata HITL，
> 及代码库实测（32 包 / 85.7k LOC / 测试比 >1:1 / 41 demo）。详见 [`COMPETITIVE-LANDSCAPE.md`](COMPETITIVE-LANDSCAPE.md)。

## 四、我们最适合哪类用户——「有需求、但今天还没被满足」

把两张矩阵叠起来看，**有人服务的格子已经挤满，没人服务的格子只有一个**：

> **需要治理 / 监护 / 审批 + 数据主权 + 跨边界协作，但① 买不起也用不上企业级平台，
> ② 又被开源框架「单组织、无治理」的天花板挡在门外的小单位。**

这个格子今天近乎真空——不是因为没人想做，而是因为它同时被**两堵墙**挡住：

- **墙 A（价格/成熟度）**：企业平台（Agentforce/ServiceNow/Microsoft/Google）有治理有 HITL，
  但是高客单价、高接触的 SaaS GTM，**根本不向下卖给家庭、奶茶店、三人律所**。
- **墙 B（架构）**：开源框架（LangGraph/Dify/n8n）和个人 agent（OpenClaw/Goose）够便宜够自托管，
  但**架构上就没有跨组织联邦、没有组织级治理、没有出站审批闸**——再便宜也长不出这些能力。

AipeHub 正好站在两堵墙之间的那条缝里：它有企业平台的「治理 + HITL + 数据主权」，又有开源框架的
「自托管 + 便宜 + 凭证只在本机」，**还独占那三列（跨组织联邦 / 个人↔组织连续 / 框架不跑 LLM）**。

### 4.1 两个最锋利、且我们已经把例子做透的滩头

| 滩头 | 是谁 | 为什么今天没被满足 | 我们已做透的例子 |
|---|---|---|---|
| **A. 家庭 / 教育** | 家长给孩子开 AI、多成员家庭主权 AI、家长监护+审批 | 企业平台不向家庭卖；个人 agent 单人单机、没有监护/跨成员主权/审批闸 | `family-learning-hub`（两主权 hub + 出站审批闸 + data-class 锁孩子数据）、`/teach` 导师、transcript fork 给家长 |
| **B. 中小企业跨组织** | 供应链（门店↔供货商）、加盟连锁（总部↔门店）、师徒/同好会/跨公司项目 | 企业平台对小微太重太贵、跨组织故事还绑厂商；开源框架是单组织 | `tea-supply-link`、`tea-chain-hq`、`warband-club`、`cafe-ops` |

往外推一圈，同一个格子还装得下**受监管的小团队联合**：律所联合、诊所、跨公司 RFP、研究协作
——都是「跨组织协作 + 数据要留在自己手里 + 要审计/审批」，今天同样无人正面服务。

### 4.2 DeepSeek 把「价格那堵墙」推倒了——这正是用户点到的变量

用户的判断完全成立：**以前没性价比的产品，以后说不定有。** 机制说清楚：

1. **这个格子过去被双重约束锁死**：①LLM 太贵 + ②没有产品填「主权+治理+跨组织+消费级价格」。
   家庭付不起、奶茶店利润薄，「每次互动都过一遍 LLM、还要养路由/会诊/心跳好几个 agent」在旧模型
   价位下**算不过账**——所以这类自托管治理型 AI 一直停在 demo，没人真上线。
2. **DeepSeek 近两个月的新 API 拆掉的是约束 ①**（LLM 成本）。**AipeHub 填的正是约束 ②**（缺的产品）。
   两件事合上，这个格子第一次同时具备「付得起」和「有东西用」。
3. **关键不对称——竞品也能用便宜的 DeepSeek，但便宜 LLM 帮不到它们够到这个格子**：
   - 企业平台：便宜 LLM 不改变它的高接触企业 GTM，它**不会**为了省 token 就下沉去卖给家庭/小店。
   - 开源框架：便宜 LLM **加不出**跨组织联邦/治理/HITL——架构缺的东西，降价补不上。
   - 个人 agent：便宜 LLM **加不出**监护/跨组织/审批——它设计上就是单人单机。
   - → DeepSeek 是普涨的潮水，但它**不成比例地解锁了 AipeHub 这一格**：因为这一格当年是被
     「成本 ∧ 缺产品」同时挡住的，DeepSeek 移走成本，而**只有 AipeHub 供得上那个缺的产品**。
4. **而且便宜 LLM 对 AipeHub 的增益比对别人更大**——这是被忽略的一点：AipeHub「框架不跑 LLM、
   但参与者跑」的设计，天然会跑**很多个小 LLM 调用**（路由 agent 决定派给谁、多 agent 会诊、心跳
   主动唤醒、三柱成长 agent、导师+主题筛查+内容审核……）。这种「多个便宜 LLM 参与者」的形态，
   在旧模型价位下是**成本负担**，恰恰把非企业用户挡在门外；DeepSeek 把单价砍下来后，AipeHub
   最自然的设计反而变成**性价比甜区**——它正好甜在过去被价格挤出去的那批用户身上。

### 4.3 为什么这个选择还顺手解了「联邦冷启动」死亡陷阱

§二说过：联邦类产品最大的死亡陷阱是**对端数＝0**。我们选的这两个滩头**自带对端**：

- 「家长 + 孩子」一开局就是 **2 个主权 hub**；
- 「门店 + 供货商」「总部 + 加盟店」「师傅 + 徒弟」一开局就是 **≥2 方**。

也就是说，这类用户的**使用场景本身就是成对/成群的联邦**——第二个对端不是要去 BD 拉来的，而是
用例自带的。这跟「企业买一套单部署」根本不同：企业单部署冷启不出联邦网络，而一对家庭、一条供应链
**天生就把第二个节点带进来了**。所以选这个格子，既是「最未被满足的需求」，又顺手把联邦的网络效应
冷启动问题从「死亡陷阱」变成「用例自带」。

### 4.4 诚实边界（这不是「躺赢」）

- **墙 A 的价格被推倒，但墙 A 的「成熟度」没有**：家庭/小店要的是**真·开箱**（一行起、桌面壳、
  傻瓜引导），AipeHub 现在还是 example-first + 要配置。性价比解锁了需求，**易用性是下一道门槛**。
- **信任/合规对家庭和受监管小团队仍是硬门槛**：监护孩子数据、律所联合，没有审计/合规背书一样不敢用。
- **分发仍是商业问题不是代码问题**：这个格子的用户散、获客难，需要无代码入口 + 模板画廊 + 真实样板户，
  不是再写几个特性能解决的。

## 五、一句话给决策用

> **最该瞄准的，是「要治理/监护/跨组织、但企业平台够不着、开源框架长不出」的小单位——
> 首推家庭/教育，其次中小企业跨组织协作。** 它们过去同时被「LLM 太贵」和「没有这个产品」挡住；
> DeepSeek 近两月的降价拆掉了前者，AipeHub 正好是后者的答案，而且这类用户的用例**自带联邦对端**，
> 顺手解了冷启动。竞品也能用便宜的 DeepSeek，但降价补不上它们架构里缺的跨组织治理——**这一格的
> 性价比窗口，结构上是给 AipeHub 开的。** 剩下的硬仗在易用性、信任背书、分发，不在技术。
