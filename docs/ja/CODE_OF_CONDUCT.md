# 行動規範

<!-- doc-version: 1.0 -->
> **ドキュメントバージョン 1.0** · 日本語訳 · 最終更新 2026-06-27 · 権威ある原典：[English](../../CODE_OF_CONDUCT.md)。訳文と英語版に矛盾がある場合は、英語版が優先されます。

AipeHub は、良い場所であり続けたいと思う小さなプロジェクトです。このドキュメントでは、私たちが期待すること、許容しないこと、そして誰かが一線を越えたときに何が起こるかを述べます。これは意図的に私たち自身の言葉で書かれています — ルールが短いので短いのです。

## 基準

このプロジェクトの人々 — イシュー、プルリクエスト、Discussions、チャット — を、悪い日にシニアエンジニアに接してほしいように接してください: 忍耐、率直さ、そして敬意を持って。コードについて激しく意見が異なっても、その人に対して親切であることはできます。

**期待すること:**

- 異なる観点、経験レベル、第一言語を尊重する。このプロジェクトには中国語と英語を話す参加者がいます。どちらの方向でも拙い英語は性格の欠陥ではありません。
- 人ではなくアイデアを批判する。「このアプローチには競合状態がある」はレビューです。「並行処理を明らかに理解していない」はそうではありません。
- 善意を推定し、それを返す。ほとんどの摩擦は悪意ではなく誤解です。
- メンテナーが「いいえ」または「今ではない」と言う可能性があること、そしてクローズされた PR は個人的な拒絶ではないことを受け入れる。

**許容しないこと:**

- あらゆる種類のハラスメント、脅迫、差別 — 年齢、体型、障害、民族性、性別特性、性自認または表現、経験レベル、国籍、外見、人種、宗教、または性的アイデンティティに基づくものを含む。
- 性的な言語や画像、または歓迎されない性的な注意。
- 個人的または政治的な攻撃、持続的な混乱、または意図的な脅迫。
- 明示的な許可なしに誰かの私的情報を公開すること。
- 専門的な場で不適切と合理的に見なされるその他の行為。

これはすべてのプロジェクトスペースに適用され、あなたがプロジェクトを公の場で代表するときにも適用されます。

## 報告

このコードに違反する行動を経験または目撃した場合は、**メンテナーに非公開で報告してください**。最初に公開でその人と対峙する義務はありません。

> **暫定的な報告チャネル（1.0 以前）:** プロジェクトにはまだ専用の行動メールボックスがありません。それまでの間、リポジトリの GitHub **プライベート報告**（Security → report）を使用してメンテナーに機密で連絡してください。報告を行動問題としてマークしてください。専用の明確にラベル付けされた行動連絡先の設立は追跡された事前リリースタスクです — 存在するときにこのセクションでそれを名指しします。

報告は機密として扱われます。報告者の同意なしに報告された人に報告者を明かしません。

## 執行

メンテナーはこのコードを守る責任があり、誠意を持って報告に対応します。重大度に応じて、対応は以下のとおりです:

1. **非公開での指摘** — 何が問題でなぜかを静かに説明する訂正。
2. **警告** — 継続した場合の結果を述べた正式な通知。
3. **一時的な禁止** — プロジェクトスペースからの時間制限付きの排除。この期間中は関係者とのやりとりなし。
4. **永久禁止** — 重大な違反、または警告後の持続的なパターン。

このコードを誠意を持って守らないメンテナー自身も、他のメンテナーとスチュワードによって決定される結果に直面する可能性があります（[`GOVERNANCE.md`](../../GOVERNANCE.md) を参照）。

問題を解決する最もシンプルな対応を優先します。目的は健全なプロジェクトであり、審判所ではありません。

## スコープと精神

このコードは意図的に軽量です。文字通りカバーしていない状況が生じた場合は、精神を適用してください: 思慮深い人はこれを敬意を持った誠意のあるものと考えるか？そうでなければ、記載されているかどうかにかかわらず、範囲外です。
