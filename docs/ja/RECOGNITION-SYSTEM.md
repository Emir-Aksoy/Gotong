# 表彰システム

<!-- doc-version: 1.0 -->
> **ドキュメントバージョン 1.0** · 日本語訳 · 最終更新 2026-06-27 · 権威ある原典：[English](../RECOGNITION-SYSTEM.md)。訳文と英語版に矛盾がある場合は、英語版が優先されます。

> このシステムが付与するのは**表彰のみ** — 金銭なし、トークンなし、報奨金なし。その
> 「通貨」は誠実な出所表記、可視化された帰属、そしてプロジェクトの方向性に実際の
> 発言権を持つための文書化された道です。

## 1. 表彰のみである理由

AipeHub の長期的な形は、テンプレート、アダプター、知識ベースコネクターからなる
**再利用可能なコンポーネントのガバナンスされたマーケット** — 人々が家庭、家族、
あるいはお金を委ねるほど信頼できるもの（[`GOVERNANCE.md`](../../GOVERNANCE.md)
§「コンポーネント委員会への道」参照）です。マーケットが生きるためには、
コントリビューターが良い作業を手渡す理由、そしてそれをメンテナンスし続ける
理由が必要です。

4 つの候補を検討し、**最初の 2 つだけを実施**します:

| 候補 | 内容 | 決定 |
|---|---|---|
| **A — FLAGSHIP 内の引用ランキング** | 「最もフォークされた人」のランキングをチェックインされたドキュメントとしてレンダリングし、静的サイトをデプロイせずにリポジトリで見られるようにする。 | ✅ 実施 |
| **B — 定量化されたメンテナーラダー** | `GOVERNANCE.md` の昇進パスに**軽量で測定可能な**基準 + `MAINTAINERS.md` を付与する。 | ✅ 実施 |
| **C — 経済的 / 報酬レイヤー** | 報奨金、トークン、収益分配。 | ❌ 廃止 |
| **D — 何もしない** | 現状維持。 | ❌ 廃止 |

**C の廃止は意図的であり、怠慢ではありません。** 北極星はフレームワークが LLM を
実行せず、状態はディスク上のファイルであり、クレデンシャルはローカルに留まり、
フェデレーションはピアツーピアであることを示しています — そして金銭を導入する
インセンティブレイヤーはその信頼モデルを直ちに曇らせます: 誰が台帳を管理するか？
ハブ間で収益分配はどう解決されるか？価格を設定する権限は誰にあるか？それらすべてが
プロジェクトを「Hub は意図的に単純で、決定は参加者にある」から離れ、中心へと引き
戻します。**純粋な表彰システムはネイティブにファイルファーストでネイティブに
分散化されています:** 帰属はテンプレートファイルの 1 つの `provenance` 行であり、
ランキングは決定論的な計算であり、昇進はパブリックイシューでのレイジーコンセンサス
— そのどれも中央の資金プールを必要としません。

このシステムの「通貨」は 3 つのものであり、どれもコストがかかりません:

1. **誠実な出所表記** — `provenance.derivedFrom` がクレジットを上流へ流します。
2. **可視化された帰属** — ランキングとフラッグシップインデックスがあなたの名前を
   最も見えやすい場所に置きます。
3. **発言権への文書化された道** — 継続的な良い作業がメンテナーの地位と実際の声を
   得ます — 支払いではありません。

これら 3 つはまた、このシステムに私たちが*追加*するあらゆるものの試金石でもあります：
コストゼロで、ファイルと人に向かうもの、中心に向かわないもの。**以下の支柱⑤ ——
普及を認める —— は一つの意図的な拡張です。** それは*可視化された帰属*を広げて、
プロジェクトを人々に届ける仕事をカバーします。元の 4 つの柱はすべてリポジトリ内の
アーティファクトをキーとしており、構造的にそれを見ることができません。それは金銭も
トラッキングバックエンドも導入しません；それは 2 つの markdown ファイルです。

## 2. 柱

このシステムは**すでに存在し、すでに接続されている**5 つのものから成ります。
これらは新しい仕組みではありません — このドキュメントは既存のパーツを 1 つの
システムとして名付け、リポジトリ外でプロジェクトを届ける仕事を認定する
一つの意図的な拡張（支柱⑤）を追加します。

### 柱① — 引用ランキング（クレジットが流れ戻る）

> 「最もリミックスされた人」は「最も有用な人」です。

すべてのテンプレートマニフェストは `provenance.derivedFrom` を持ちます。
テンプレートをフォークするとき、上流のスラッグを自分の `derivedFrom` に書き込みます。
ランキングは**入次数** — 何個のテンプレートがあなたから派生していると宣言しているか —
で順位付けします。

- **仕組み**: `packages/web/scripts/build-site.mjs` の純粋関数 `loadCorpus` +
  `buildModel` が検証済みコーパスから入次数を計算します。
- **1 つの計算で 2 つのレンダリングターゲット**:
  - 静的サイト（[`zh/COMMUNITY-SITE.md`](../zh/COMMUNITY-SITE.md)）がレンダリングします;
  - **チェックインされたドキュメント**（[`zh/FLAGSHIP-TEMPLATES.md`](../zh/FLAGSHIP-TEMPLATES.md)
    の「引用ランキング」セクション）もレンダリングします — これが**柱 A** であり、
    `pnpm build:leaderboard`（`build-leaderboard-doc.mjs`）によって
    `<!-- LEADERBOARD:START -->` マーカーブロックに書き込まれます。静的サイトを
    デプロイせずにリポジトリでランキングを確認できます。
- **ドリフトガード**: `packages/web/tests/build-leaderboard-doc.test.ts` が
  実際のコーパスから再レンダリングし、チェックインされたブロックがバイト同一で
  あることを検証します — `derivedFrom` エッジを追加して `pnpm build:leaderboard`
  の再実行を忘れると、テーブルが静かに腐る代わりに CI が失敗を指摘します。
- **テンプレートをランク付けし、人々ではない。** これが重要な誠実な境界です:
  ランキングは*コンポーネント*の再利用量を測定します; パーソナリティカルトを動かし
  たり、ゲーム可能なヘッドカウントポイントを発行したりしません。

### 柱② — メンテナーラダー（発言権への道）

> 良いコントリビューションの終点は**信頼 + 責任**であり、賞品ではありません。

`GOVERNANCE.md` の「メンテナーになる」は**意図的に軽量で測定可能な**基準を
与えます（これが**柱 B**）:

- **集計ではなく実績**: 数ヶ月にわたっておよそ 5 つの非自明なマージ済み PR — または
  同等のもの（メンテナンスし続けるフラッグシップテンプレート、実質的なアダプター、
  継続的なレビュー / トリアージ）。この数字は「あなたの判断を十分見てきた」という
  **フロア**であり、ドライブバイ PR で農業する**目標**ではありません。
- **設計ラインへの感覚**: あなたの PR とレビューが、ロジックの居場所が必要なとき
  Hub ではなく*参加者*に向かうことを示します（`GOVERNANCE.md` § 「唯一の譲れない
  もの」参照）。
- **公開で指名される**: 既存のメンテナーがパブリックイシューであなたを指名します
  （自己指名も可）; レイジーコンセンサスが通過し、スチュワードが確認し、あなたの
  名前が同じ PR で [`MAINTAINERS.md`](../../MAINTAINERS.md) に記載されます。

`MAINTAINERS.md` は今日、創設メンテナーのみを保持しています。そのファイルの
全ポイントは、**2 番目の**メンテナーが肩叩きではなく**書き下された**道によって
参加することです — 責任の流れは暗黙の習慣であってはなりません。コントリビューションの
量がキュレーションを定常的な仕事にするほど大きくなったとき、`GOVERNANCE.md` は
すでに**コンポーネント委員会**を設立する計画を記録しています。

### 柱③ — 摩擦のない共有（手渡しを安くする）

> 摩擦はインセンティブの敵です。テンプレートのインストールはワンクリック;
> 提出は 20 ステップであるべきではありません。

- **ワンクリックインストール**: 管理者「ワークフロー」パネルの**テンプレートギャラリー**
  （[`zh/TEMPLATE-GALLERY.md`](../zh/TEMPLATE-GALLERY.md)）がフレームワークに
  同梱された厳選テンプレートを一覧し、既存の `POST /templates/import` を再利用した
  ワンクリックインストールができます。
- **5 ステップ提出**: コミュニティテンプレートの提出フローは
  [`templates/community/templates/README.md`](../../templates/community/templates/README.md)
  にあります — フラッグシップをコピー → 自分のものにする → 出所を宣言する →
  `pnpm check:templates` をローカルで実行する → PR を開く。
- **バーは安全性と誠実さであり、好みではない**: コミュニティティアは「ライセンスが
  明確、解析可能、プレーンテキストのシークレットがゼロ、出所が宣言済み」のみを
  求めます（`GOVERNANCE.md` § 「コミュニティテンプレート」）; それを満たせば
  マージされます。このティアでは*安全性と誠実さ*をキュレートします、好みではありません。

利便性はそれ自体がインセンティブです: テンプレートを手渡しする費用が低ければ低いほど、
より多くの人々が個人的に抱え込んでいた良いワークフローを公開します — そして
すべての誠実で帰属された公開が上流に 1 つの柱①引用を与えます。

### 柱④ — 共有の手本（リミックスする価値のあるもの）

> ランキングには引用するものが必要; まずは引用に値する手本が必要です。

- **フラッグシップティア**: [`zh/FLAGSHIP-TEMPLATES.md`](../zh/FLAGSHIP-TEMPLATES.md)
  — プロジェクトが保証し非技術ユーザーに推薦する小さな厳選セット。バーは高い
  （決定論的デモ + 明確なガバナンス姿勢 + メンテナー、`GOVERNANCE.md` §
  「フラッグシップテンプレート」参照）。
- **組み込みギャラリー**: フレームワークに組み込まれたテンプレート、管理 UI から
  ワンクリックでインストール可能。
- **examples/**: エンドツーエンドデモ、それぞれフォーク可能なスタートポイント。

手本はループの**種**です: 良い手本なしでは、摩擦のない共有には何も共有するものが
なく、ランキングには何もランク付けするものがありません。手本を良く書き、そのガバナンス
姿勢を明確に述べると、人々がそれをフォークし、引用し、その上に自分の作業を積み
上げます。

<a id="pillar-5"></a>

### 支柱⑤ — 普及の認定（届ける仕事）

> 「最も有用なフレームワーク」は使われるものです。プロジェクトを人々に届ける仕事は、
> コードと同じくらいそれを実現します。

支柱①から④はすべてリポジトリ内のアーティファクトをキーとしています：テンプレート、
PR、レビュー。それらはチュートリアルを書いたり、トークをしたり、ドキュメントを
翻訳したり、新参者が詰まらなくなる場を運営する人 — リポジトリ内に痕跡を残さない
かもしれないが、プロジェクトを実際に人々に届ける人 — を構造的に見ることができません。

支柱⑤はその仕事を 2 つの**ファイルファースト**のアーティファクトで認定します：

- **[`CONTRIBUTORS.md`](CONTRIBUTORS.md)** — 手動でメンテナンスされる型付き
  コントリビューターレジャー、[All Contributors](https://allcontributors.org)
  の emoji 語彙を使用（ただしその bot または GitHub Action は**使いません**）。
  コードを書くこと、ビデオを作ること、翻訳をすること、コミュニティを運営すること
  —— これらはすべて**同じテーブル**に記録されます。通常の pull request で
  追加されます。
- **[`LEARN.md`](LEARN.md)** — キュレーションされた学習陳列館（ビデオ、
  トーク、チュートリアル）。視聴数ではなく質が基準です：正確で、有用で、
  完成しており、帰属表示されていること。掲載は編集上の認定であり、
  著者を数字でランク付けしません。

**どちらも金銭なし、bot なし、トラッキングバックエンドなし。** それらは
markdown ファイルであり、pull request で更新され、git で管理されます —— これで
十分です。

## 3. 柱が互いを強化する方法

五つの柱は 5 つの孤立したものではありません — それらは**自己強化ループ**であり、
支柱⑤はそれを外への届ける仕事にまで広げます:

```
   ④ 手本  ──フォーク──▶  ③ 摩擦のない共有  ──PR + 誠実な出所表記──▶  ① ランキング
        ▲                                                                    │
        │                                                    引用 = 可視化された帰属
        │                                                                    │
        └──────────────  継続的な良い作業  ◀──②  メンテナーラダー  ◀──────────┘
                       （新しい手本 / 古いものの維持 / 他者のレビュー）
```

1. **フラッグシップの手本（④）** から始めます;
2. それを自分のものにし、**摩擦のない共有（③）**を通じて手渡し、
   `provenance` で上流を**誠実に帰属**します;
3. あなたの誠実な出所表記が上流に 1 つの**引用（①）**を加え、ランキングを登ります
   — クレジットが流れ戻ります;
4. あなた自身のテンプレートがフォークされ引用され始め、あなたの名前がランキングと
   フラッグシップインデックスに載ります;
5. 継続的な良い作業（新しい手本、古いものの維持、他者のレビュー）が
   **メンテナーラダー（②）**を登り、信頼と声を得ます — そしてメンテナーとして
   新しいフラッグシップの手本（④）を保証し、ループが再び回ります。

**どのステップも金銭を必要としません。** ループ全体を動かすのは「私のものが有用で、
人々がそれを使い、私の名前が公開されており、私の言葉が重みを持ち始める」— 純粋な
表彰であり、それで十分です。

**支柱⑤はどこに位置するか。** 支柱⑤はループの外側ではなくその**入口**です。
チュートリアルが新参者をプロジェクトに連れてくると、その新参者たちが手本（④）を
見つけ、フォークし（③）、引用（①）し、良い作業を続けて最終的にラダー（②）に
乗ります。届ける仕事なしに、ループは見つけられる人がいません。それは別のループでは
なく —— それはメインループのための**到達層**です。

## 4. しないこと（誠実な境界）

- **金銭なし / トークンなし / 報奨金なし**（候補 C、廃止）。
- **ランキングは人々をランク付けしない**: テンプレートがどれだけ再利用されるかを
  ランク付けし、ゲーム可能な個人ポイントではありません。
- **昇進は自動ではない**: 〜5 PR はフロアであり、スイッチではありません; 最終
  判断は人間の判断 + パブリックイシューでのレイジーコンセンサスであり、アンロック
  するカウンターではありません。
- **普及の認定は視聴者数を測定しない**: `LEARN.md` は質をキュレートします ——
  正確で有用な作品 —— ビュー数やフォロワー数ではありません。`CONTRIBUTORS.md`
  は*何をしたか*を述べます、*どれだけの人がそれを見たか*ではありません。視聴者
  規模は操作可能であり、追い求めることはプロジェクトを虚栄に引っ張ります。
- **そのために発明された新しい仕組みはない**: 五つの柱はすべて**すでに存在し、
  すでに接続されている**ものです; 支柱⑤は正確に 2 つの手動でメンテナンスされる
  markdown ファイルを追加し、それ以上のものはありません。
