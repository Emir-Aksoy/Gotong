# コントリビューター

<!-- doc-version: 1.0 -->
> **ドキュメントバージョン 1.0** · 日本語訳 · 最終更新 2026-06-27 · 権威ある原典：[English](../../CONTRIBUTORS.md)。訳文と英語版に矛盾がある場合は、英語版が優先されます。

このファイルは AipeHub を作り上げる人々を記録します —— そして**あらゆる種類の
貢献がカウントされます。大きくても小さくても、コードだけではありません。** ついに
federation を理解させてくれるチュートリアルを書くこと、五十人をプロジェクトに
連れてくるトークをすること、コアチームが話せない言語にドキュメントを翻訳すること、
新参者が詰まらなくなる場を運営すること —— それは*誰も見つけない良いフレームワーク*
を*実際に人が使う良いフレームワーク*に変える仕事であり、ここではマージされた PR
と**同等の認定**を得ます。**記録するには小さすぎる努力はなく、重要な普及努力は
リリースされた機能と同じようにこの記録に残ります。**

これは[表彰システムの支柱 ⑤](RECOGNITION-SYSTEM.md#pillar-5)の人に向けた半分です。
その通貨はプロジェクト唯一の通貨です：**誠実で、見える帰属表示** ——
お金なし、トークンなし、報奨金なし。そして**記録であり、ランキングではありません。**
*あなたが何をしたか*を公開の場で、あなたの名前をつけて述べます。人を数字で並べ替えません。
なぜなら視聴者数は操作可能であり、それを追い求めることはプロジェクトを虚栄に
引っ張るからです —— [やらないこと](RECOGNITION-SYSTEM.md)を参照してください。

## 貢献の記録方法

[All Contributors](https://allcontributors.org) の emoji 語彙を、貢献の種類に
ついての共有された認識されている言語として使います —— しかしそのbot や GitHub Action
は**使いません**。このリポジトリは記帳に Actions の予算を使わず、手動でメンテナンス
される markdown テーブルが最も軽く、機能する誠実なやり方です。通常の pull request
によって追加されます。

| Emoji | 種類 | Emoji | 種類 |
|---|---|---|---|
| 💻 | コード | 📹 | ビデオ |
| 📖 | ドキュメント | 📢 | トーク |
| 🌍 | 翻訳 | ✅ | チュートリアル |
| 🎨 | デザイン | 💬 | 質問への回答 / コミュニティサポート |
| 👀 | Review | 📋 | イベント運営 |
| 🤔 | アイデアと計画 | 📝 | ブログ投稿 / ライティング |
| 🐛 | バグレポート | 🚇 | インフラ |
| 🚧 | メンテナンス | 🔌 | アダプター / コネクター |

四つのリーチ種別 —— 🌍 翻訳、📹 ビデオ、📢 トーク、✅ チュートリアル、📝 ブログ、
💬 コミュニティ、📋 イベント —— は意図的に 💻 コードと**同じテーブル**に置かれています。
支柱 ⑤ の要点はそれらが劣ったコラムではないということです。

## 追加される方法

1. **pull request を開いて**行を追加してください —— あなた自身のものでも、他の人の
   ものでも。他の人の仕事を認定することは歓迎され、奨励されます。
2. **基準は上の凡例の任意の種類の、本物の誠実な貢献です。** コード/ドキュメントの
   場合、PR 自体が証拠です。**リーチ**貢献（ビデオ、トーク、チュートリアル、
   投稿、翻訳、または継続的なコミュニティサポート）の場合、maintainer が見られるよう
   PR の説明に**成果物へのリンクを付けてください**。
3. **maintainer がマージします** —— 小さな PR と同じ方法で（lazy consensus、
   [`GOVERNANCE.md`](../../GOVERNANCE.md#how-a-change-lands) 参照）。bot なし、儀式なし。

あなたの貢献が**学習教材** —— AipeHub をうまく教えるビデオ、チュートリアル、または
トーク —— であれば、新参者が案内される陳列館 [`LEARN.md`](LEARN.md) にも
キュレーションされることができます。ここに記録されることとそこにキュレーションされることは
二つの異なる認定です；優れたチュートリアルは両方を得るべきです。

## Standing（一行の記録を超えて）

継続的なリーチワークはここに記録されるだけでなく —— プロジェクトにおける
実質的な発言権を得ます。`GOVERNANCE.md` の [maintainer
への昇格路径](../../GOVERNANCE.md#becoming-a-maintainer) はコードに対する
**等価なトラック**としてカウントします：複数の言語でドキュメントを生き続けさせること、
コミュニティを運営すること、継続的な教育材料。フレームワークのコードを一行も書かない
けれど週次でその仕事をする人は、まさにその昇格路径が認定しようとしている種類の
継続的な判断力を貢献しています。
