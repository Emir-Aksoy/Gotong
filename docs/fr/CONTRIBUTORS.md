# Contributeurs

<!-- doc-version: 1.0 -->
> **Version du document 1.0** · Traduction française · Mis à jour le 2026-06-27 · Source faisant autorité : [English](../../CONTRIBUTORS.md). En cas de divergence entre la traduction et la version anglaise, la version anglaise prévaut.

Ce fichier recense les personnes qui construisent AipeHub — et **chaque type de
contribution compte, grande ou petite, pas seulement le code.** Écrire le tutoriel
qui fait enfin comprendre la fédération, donner la conférence qui amène cinquante
personnes au projet, traduire la documentation dans une langue que l'équipe centrale
ne parle pas, tenir la salle où les nouveaux se débloquent — c'est le travail qui
transforme *un bon framework que personne ne trouve* en *un bon framework que les gens
utilisent vraiment*, et il est reconnu ici avec le même standing qu'un PR fusionné.
**Aucun effort n'est trop petit pour être enregistré, et un effort significatif de
diffusion atterrit dans ce registre au même titre qu'une fonctionnalité livrée.**

C'est la moitié tournée vers les gens du [pilier ⑤ du système de
reconnaissance](RECOGNITION-SYSTEM.md#pillar-5).
Sa monnaie est la seule monnaie du projet : **une attribution honnête et visible** —
pas d'argent, pas de jeton, pas de prime. Et c'est **un registre, pas un classement.**
Il indique *ce que vous avez fait*, au grand jour, avec votre nom dessus. Il ne classe
pas les gens selon un chiffre, parce que la taille d'audience est manipulable et la
poursuivre tire un projet vers la vanité — voir
[Ce que nous ne faisons pas](RECOGNITION-SYSTEM.md).

## Comment une contribution est enregistrée

Nous utilisons le vocabulaire emoji de [All Contributors](https://allcontributors.org)
comme langage partagé et reconnu pour les types de contributions — mais **pas** son bot
ou son GitHub Action. Ce dépôt ne dépense pas de budget Actions pour la comptabilité,
et une table markdown maintenue à la main est la chose la plus légère et honnête qui
fonctionne. Vous êtes ajouté via un pull request ordinaire.

| Emoji | Type | Emoji | Type |
|---|---|---|---|
| 💻 | Code | 📹 | Vidéo |
| 📖 | Documentation | 📢 | Conférence |
| 🌍 | Traduction | ✅ | Tutoriel |
| 🎨 | Design | 💬 | Réponses aux questions / support communautaire |
| 👀 | Review | 📋 | Organisation d'événements |
| 🤔 | Idées & planification | 📝 | Billet de blog / écriture |
| 🐛 | Rapports de bugs | 🚇 | Infrastructure |
| 🚧 | Maintenance | 🔌 | Adapter / connecteur |

Les quatre types de diffusion — 🌍 traduction, 📹 vidéo, 📢 conférence, ✅ tutoriel,
📝 blog, 💬 communauté, 📋 événements — siègent délibérément dans la **même table**
que 💻 code. Tout l'intérêt du pilier ⑤ est qu'ils ne constituent pas une colonne
inférieure.

## Comment être ajouté

1. **Ouvrez un pull request** en ajoutant une ligne — la vôtre ou celle de quelqu'un
   d'autre. Créditer le travail d'une autre personne est bienvenu et encouragé.
2. **Le critère est une contribution réelle et honnête** de n'importe quel type dans
   la légende ci-dessus. Pour le code/la documentation, le PR lui-même est la preuve.
   Pour une contribution de **diffusion** (une vidéo, conférence, tutoriel, article,
   traduction, ou aide communautaire soutenue), **liez l'artefact** dans la description
   du PR pour qu'un maintainer puisse le voir.
3. **Un maintainer le fusionne** de la même façon que tout petit PR se fusionne (lazy
   consensus, voir [`GOVERNANCE.md`](../../GOVERNANCE.md#how-a-change-lands)). Pas de
   bot, pas de cérémonie.

Si votre contribution est un **matériel d'apprentissage** — une vidéo, un tutoriel ou
une conférence qui enseigne bien AipeHub — il peut *aussi* être mis en valeur dans
[`LEARN.md`](LEARN.md), la vitrine vers laquelle les nouveaux sont orientés. Être
enregistré ici et être mis en valeur là-bas sont deux actes de reconnaissance distincts ;
un excellent tutoriel mérite les deux.

## Standing, pas seulement une ligne

Un travail de diffusion soutenu n'est pas seulement enregistré ici — il donne une vraie
voix dans le projet. Le [parcours de maintainer](../../GOVERNANCE.md#becoming-a-maintainer)
dans `GOVERNANCE.md` compte un **parcours équivalent** au code : maintenir la
documentation vivante dans plusieurs langues, animer la communauté, du matériel
éducatif soutenu. Une personne qui n'écrit jamais une ligne de code du framework mais
fait ce travail semaine après semaine contribue exactement le type de jugement soutenu
que le parcours est censé reconnaître.
