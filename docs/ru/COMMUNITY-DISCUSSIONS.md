# GitHub Discussions — «гостиная» сообщества (нулевые вычисления, однократное включение)

<!-- doc-version: 1.0 -->
> **Версия документа 1.0** · Перевод на русский язык · Обновлено: 2026-06-27 · Авторитетный источник: [English](../COMMUNITY-DISCUSSIONS.md). При расхождении перевода с английской версией приоритет имеет английская версия.

> Пункт 8 контрольного списка перед запуском. Одна строка: **Issues — это стойка регистрации, Discussions — гостиная** — вопросы, демонстрации результатов и предложение идей — всё это происходит здесь; GitHub хостит это бесплатно, **нулевые вычисления**, как и лендинг/таблица лидеров.

---

## 1. Почему Discussions (а не ещё один сервис)

Та же позиция, что и в [`COMMUNITY-SITE.md`](COMMUNITY-SITE.md): для проекта, ориентированного на файлы, чей хаб не запускает LLM сам, **инфраструктура сообщества тоже не должна нуждаться в сервере**. GitHub Discussions хостит всю «гостиную» — треды, категории, @упоминания, Markdown, поиск — всё это работа GitHub, без единой строки нашего бэкенда.

- **Issues** = стойка регистрации для «что-то сломано / чего-то не хватает» (закрываемые, назначаемые, с состоянием).
- **Discussions** = гостиная для «хочу спросить / показать / пообщаться» (открытые, с голосованием, можно пометить лучший ответ).

Эти два входа уже направлены в [`.github/ISSUE_TEMPLATE/config.yml`](../.github/ISSUE_TEMPLATE/config.yml) — при открытии issue ссылка на контакт «💬 Вопрос или обсуждение» отправляет людей в Discussions. **Так что до включения Discussions эта ссылка даёт 404**; как только она включена, она сразу начинает работать.

---

## 2. ⚠️ Единственное ручное действие: включить Discussions (Claude не может помочь)

**Включение Discussions — это переключатель в настройках репозитория, а не файл — ни Claude, ни CI не могут его переключить.** Этот шаг должен выполнить владелец репозитория в веб-интерфейсе:

1. Откройте `https://github.com/Emir-Aksoy/Gotong/settings` (**Settings** репозитория).
2. Прокрутите вниз до раздела **Features**, поставьте галочку **Discussions**.
3. GitHub **автоматически создаст категории по умолчанию**: Announcements / General / **Ideas** / Polls / **Q&A** / **Show and tell**. Три шаблона форм, поставляемых с этим репозиторием (см. §4), нацелены на три выделенных жирным и прикрепляются автоматически **в момент** включения — без необходимости создавать категории вручную.

> Это означает «каркас готов, не хватает только переключателя»: файлы шаблонов, черновик приветственного поста, ссылка маршрутизации проблем и документация — всё это уже в репозитории; вы нажимаете Features → Discussions, и гостиная открывается.

После включения рекомендуются ещё два действия (всё это несколько кликов в веб-интерфейсе, необязательно, но рекомендуется):

- **Закрепить приветственный пост**: опубликуйте черновик из §5 как Discussion в категории General и нажмите «Pin».
- **(Необязательно) добавить пользовательскую категорию «Templates»**: если обмен шаблонами вырастает из Show and tell, создайте отдельную; но на первое время Show and tell по умолчанию достаточно — не добавляйте преждевременно.

## 3. Карта категорий (три, которые готовы с фреймворком)

| Категория | slug | Форма | Для чего |
|---|---|---|---|
| **Q&A** | `q-a` | [`q-a.yml`](../.github/DISCUSSION_TEMPLATE/q-a.yml) | Помощь, вопросы. Можно пометить «лучший ответ». |
| **Ideas** | `ideas` | [`ideas.yml`](../.github/DISCUSSION_TEMPLATE/ideas.yml) | Предлагать функции / направления. Форма подталкивает к согласованию с северной звездой (хаб не запускает LLM / file-first / peer-to-peer федерация). |
| **Show and tell** | `show-and-tell` | [`show-and-tell.yml`](../.github/DISCUSSION_TEMPLATE/show-and-tell.yml) | Показывать ваш хаб / рабочий процесс / шаблон. **Удобно направляет к отправке шаблона в галерею** + написанию `derivedFrom`, чтобы заслуга вернулась. |
| Announcements | `announcements` | — | Только для мейнтейнеров (выпуски, крупные изменения). Без формы. |
| General | `general` | — | Приветственный пост + некатегоризированная беседа. Без формы. |

**Slug = имя файла**: GitHub прикрепляет форму из `.github/DISCUSSION_TEMPLATE/<slug>.yml` к одноимённой категории. Эти три slug — категории по умолчанию, которые GitHub **автоматически создаёт** при включении, поэтому шаблоны работают «из коробки» без необходимости вручную создавать категории и сначала согласовывать имена.

## 4. Шаблоны форм (`.github/DISCUSSION_TEMPLATE/`)

Тот же подход, что и в [`.github/ISSUE_TEMPLATE/`](../.github/ISSUE_TEMPLATE/) — структурированные формы, которые заставляют автора сразу давать полезную информацию. Три шаблона, каждый с фокусом:

- **`q-a.yml`** — направляет к указанию «что вы пытаетесь сделать» (не просто ошибку) + «что вы пробовали» + версия + режим запуска; и перенаправляет **баги обратно в Issues, проблемы безопасности в SECURITY.md** — гостиная не принимает эти два.
- **`ideas.yml`** — спрашивает «в чём проблема» перед «что вы хотите», и заставляет автора предложения **самостоятельно взвесить соответствие трёхуровневой северной звезде** (всё, что требует от хаба запускать LLM / скрывать состояние / централизовывать учётные данные, — скажите честно — это не вето, но это формирует дискуссию).
- **`show-and-tell.yml`** — помимо показа результатов, **выдвигает на первый план руководство "может ли это попасть в галерею одним кликом"**: ссылается на [процесс отправки шаблона сообщества](../templates/community/templates/README.md), собирает `slug` и `derivedFrom` (питая таблицу лидеров цитирования), и превращает два жёстких правила галереи (учётные данные должны быть `${ENV}`, без контента знаний/персонала) в флажки.

> Поля форм на английском — согласно существующей конвенции `.github/ISSUE_TEMPLATE/`; вводный блок каждой формы добавляет однострочную подсказку на китайском для пользователей, предпочитающих китайский язык. Приветственный пост (§5) — сначала по-китайски, затем по-английски.

## 5. Черновик приветственного / закреплённого поста (готов к копированию)

После включения Discussions **скопируйте весь блок ниже**, опубликуйте новый Discussion в категории **General** с заголовком `👋 欢迎来到 Gotong 客厅 / Welcome` и нажмите **Pin**. Оригинальный черновик начинается с китайского (основная аудитория сообщества), затем английский; переставьте по своему усмотрению.

```markdown

- **🙋 A question?** Open one in **Q&A**. Say what you're trying to do and what you
  tried; someone will help.

  import-and-run, submit it to the one-click gallery via the
  [submit flow](../../tree/main/templates/community/templates).
- **💡 An idea?** Pitch it in **Ideas**. Gotong has a deliberate spine — aiming with
  it lands better: **the hub never runs an LLM · people and agents are the same
  Participant · state is files on disk · federation is peer-to-peer (workflows can
  cross org lines, but credentials/data/billing each stay home)**.
- **🐞 A bug?** That goes to [Issues](../../issues/new/choose), not here.
- **🔐 A security issue?** Please do **not** post it publicly — use the private

New here? Start with the [5-minute overview](../../blob/main/docs/OVERVIEW.md) and the
```

## 6. Как это связано с остальным

Этот пункт не изолирован — он связывает гостиную с линиями, которые уже проложил контрольный список перед запуском:

- **Маршрутизация проблем**: ссылка «💬 Вопрос или обсуждение» в [`ISSUE_TEMPLATE/config.yml`](../.github/ISSUE_TEMPLATE/config.yml) давно указывает на `/discussions`; как только Discussions включена, эта ссылка перестаёт быть 404.
- **Галерея шаблонов / таблица лидеров**: форма Show & Tell направляет авторов шаблонов к [процессу отправки шаблона сообщества](../templates/community/templates/README.md); после слияния отправки она появляется в галерее одним кликом ([`TEMPLATE-GALLERY.md`](zh/TEMPLATE-GALLERY.md)) и статической витрине ([`COMMUNITY-SITE.md`](COMMUNITY-SITE.md)); `derivedFrom`, собранный формой, питает таблицу лидеров цитирования.
- **Управление**: [`GOVERNANCE.md`](../GOVERNANCE.md) перечисляет Discussions как один из входов для участников; направления, сформировавшиеся в Ideas, попадают через процесс принятия решений GOVERNANCE.

Пункт «Enable GitHub Discussions» в `.github/RELEASE-CHECKLIST.md` теперь указывает на этот документ.

## 7. Ограничения (честно)

- **Claude не может включить Discussions**: это переключатель в Settings репозитория (§2), только владелец может нажать его в веб-интерфейсе. То, что может сделать этот репозиторий — «каркас»: шаблоны форм, черновик приветственного поста, ссылка маршрутизации, документация — всё готово.
- **Формы не являются проверкой**: шаблоны Discussion только **направляют публикацию**, они не блокируют и не валидируют. Реальная валидация для шаблона, попадающего в галерею, — это [`pnpm check:templates`](../templates/community/templates/README.md) (проходящий настоящий `parseTemplate`), что является отдельным вопросом.
- **Нет принудительной миграции истории**: ссылки, разбросанные по документации сегодня, указывающие на `/discussions` (REAL-WORLD-TESTING, LICENSE-FAQ и т.д.), оживают естественным образом после включения, без необходимости возвращаться и редактировать их.
