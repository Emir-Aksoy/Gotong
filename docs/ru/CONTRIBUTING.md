# Участие в разработке AipeHub

> **Версия документа 1.0** · Перевод на русский язык · Обновлено: 2026-06-27 · Авторитетный источник: [English](../../CONTRIBUTING.md). При расхождении перевода с английской версией приоритет имеет английская версия.

Спасибо, что рассматриваете возможность внести вклад. AipeHub — проект на ранней стадии, и мы рады принимать патчи, отчёты об ошибках, обратную связь по дизайну, улучшения документации и работу по распространению — видео, доклады, переводы, поддержку сообщества. Весь вклад записывается в [`CONTRIBUTORS.md`](../../CONTRIBUTORS.md).

## Основные правила

- **Будьте добры.** Относитесь к любому в трекере задач / PR так, как бы вы хотели, чтобы старший инженер относился к вам в плохой день.
- **Маленькие PR.** Независимые изменения выходят быстрее, чем мега-PR. Если функция разделяется чисто, отправляйте части отдельно.
- **Hub остаётся простым.** Вся идея дизайна AipeHub в том, что Hub маршрутизирует / сохраняет и не владеет логикой агентов. Патчи, добавляющие вызовы LLM, циклы агентов или бизнес-правила в Hub, будут перенаправлены.
- **Проводной протокол версионирован.** Всё, что меняет форму сообщений на уровне протокола, проходит через `docs/PROTOCOL.md` и повышение версии протокола. Локальные изменения — нет.
- **Никаких неожиданных зависимостей.** Добавление runtime-зависимости (особенно нативных) — это настоящее решение — сначала откройте issue.

## Workflow

```bash
# форкните на GitHub, затем:
git clone git@github.com:<you>/AipeHub.git
cd AipeHub
pnpm install
pnpm build

# вносите изменения…

pnpm -r typecheck      # все 19+ пакетов проходят проверку типов без ошибок
pnpm -r test           # vitest по всем пакетам
pnpm test:python       # python-sdk pytest
```

Соглашения:

- TypeScript strict mode, ESM с расширениями `.js` в относительных импортах (разрешение TypeScript «node16/nodenext» требует этого).
- Тесты живут рядом с кодом, который они покрывают (`packages/*/tests/`).
- Линтинг пока не применяется инструментом; соответствуйте стилю существующих файлов.
- Сообщения commit: повелительное наклонение («add foo», а не «added foo»). Один абзац для нетривиальных commit'ов приветствуется.

## Структура репозитория

```
packages/
  core/           Hub + реестр + планировщик + transcript + Space
  protocol/       Типы проводного протокола (без среды выполнения)
  transport-ws/   WebSocket-адаптер на стороне Hub
  sdk-node/       Node SDK для удалённых агентов (connect + AgentParticipant)
  web/            Встраиваемый веб-сервер + статичный SPA
  host/           Производственный бинарный файл (управляемый переменными окружения, без демонстрационного состояния)
  llm/            Базовый класс LlmAgent + интерфейс LlmProvider
  llm-anthropic/  Провайдер Anthropic
  llm-openai/     Провайдер OpenAI
python-sdk/       Python SDK (зеркало sdk-node)
examples/         Запускаемые демонстрации
docs/             Подробная документация по архитектуре / протоколу / развёртыванию
```

## Области для работы

Если вы хотите задачу с низким порогом входа, ищите issues с меткой `good-first-issue`. Некоторые всегда актуальные темы:

- **Документация**: опечатки, более чёткие примеры, переводы (у проекта есть мейнтейнеры, говорящие по-китайски; документация только на английском пока тоньше).
- **Покрытие тестами**: особенно для граничных случаев планировщика и путей миграции Space на диске.
- **Дополнительные провайдеры LLM**: скопируйте форму `packages/llm-anthropic`.
- **A11y / i18n в admin UI**: vanilla JS, без фреймворка, небольшая поверхность.

## Участие в создании шаблонов

Не обязательно писать TypeScript, чтобы внести вклад. AipeHub поставляет **шаблоны** — автономный YAML, который кто-то импортирует для получения рабочего hub'а (агенты + workflows + ссылки на базу знаний, никогда не секреты или содержимое знаний).

- Одна адаптированная подсказка → [`templates/community/`](../../templates/community/).
- Целый импортируемый hub (мультиагентный + workflows) → [`templates/community/templates/`](../../templates/community/templates/) — в этом README описан 5-шаговый процесс: скопировать флагманский пример, адаптировать его, объявить происхождение (`derivedFrom`), проверить локально с помощью `pnpm check:templates`, открыть PR.

Планка для *слияния в качестве шаблона сообщества* (лицензия чистая, парсится, нет литеральных секретов) ниже, чем планка для *поставки как флагманского* (детерминированная демонстрация, заявленная позиция управления, поддерживается). Смотрите [`GOVERNANCE.md`](../../GOVERNANCE.md).

## Распространение тоже считается

Не обязательно писать код, чтобы существенно помочь. Работа по распространению — перевод документации, видеоурок, доклад, поддержка сообщества, организация встречи — ценится наравне с кодом; её трудно сделать, и без неё хорошие фреймворки остаются незамеченными.

- **Создали видео или обучающий материал?** Добавьте его в [`LEARN.md`](../../LEARN.md) — это кураторская витрина, которую проект рекомендует новичкам. Откройте PR с вашей записью в нужном разделе; maintainer проверит по [критериям](../../LEARN.md#the-bar-we-curate-quality-not-view-count) и смёрджит.
- **Перевели документ или поддерживаете перевод?** Это **первоклассная работа** — поддерживать документацию на языке, которым основная команда не владеет, — именно такой устойчивый вклад, который открывает [лестницу maintainer'а](../../GOVERNANCE.md#becoming-a-maintainer).
- **Помогаете новичкам в Discussions, на форумах или в группах?** Устойчивая поддержка сообщества тоже записывается как вклад.

Работа по распространению не ранжируется по охвату аудитории. Материал попадает в `LEARN.md` потому что он **точный и полезный**, а не потому что у него много просмотров. Запись в `CONTRIBUTORS.md` — за честный вклад, а не за популярность.

## Сообщение об ошибках

Полезный отчёт об ошибке содержит:

- Что вы пробовали (полная командная строка, полные переменные окружения)
- Что вы ожидали
- Что произошло (полный вывод ошибки, если есть, фрагмент `transcript.jsonl` если ошибка в маршрутизации / сохранении)
- Версии: `node --version`, `pnpm --version`, ОС

Для ошибок, связанных с сетевой формой (workers отключаются, агенты не маршрутизируются), включите снимок `/api/state` — это канонический «что, по мнению hub'а, происходит».
