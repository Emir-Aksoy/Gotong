[English](../../README.md) · [中文文档](../zh/README.md)

**ИИ + Человек + Hub** — самостоятельно развёртываемая основа, в которой люди и агенты ИИ взаимодействуют как равноправные участники, а организации образуют федерацию, не передавая свои ключи, данные или биллинг.

Gotong — это не агент и не ещё один фреймворк для агентов. Это **уровень под ними**: реестр, шина сообщений, маршрутизатор задач, управляемый федеративный канал и неизменяемый transcript. Агенты LangGraph / CrewAI, CLI-агенты для написания кода (Claude Code, Codex) и люди подключаются как одинаковый `Participant`. Hub поддерживает сигналы в движении и соблюдает границы — он никогда не запускает LLM, поэтому все решения остаются за участниками.

### ИИ, которому действительно можно доверить важное

Большинство инструментов ИИ предлагают два варианта: отдать всё в облако, которым вы не управляете, или самостоятельно соединить всё вместе. Gotong — третий вариант: **ИИ, которому можно доверить ваш дом, вашу семью или ваши деньги, потому что границы реальны и принадлежат вам:**

- **Человек в контуре там, где это важно.** Обратимые действия (выключить свет) просто выполняются; необратимые (заблокировать дверь, потратить деньги, передать данные ребёнка по каналу) ждут, пока человек не подтвердит в inbox. Workflow не может обойти ворота.
- **Ваши ключи и данные остаются на вашем диске.** Учётные данные хранятся зашифрованными в вашей собственной директории `.gotong/`. Федерация с другим hub'ом предоставляет доступ к возможности, а не к вашему vault.
- **Ничто не решается в темноте.** Каждая отправка и результат записываются в доступный для чтения неизменяемый transcript. Фреймворк никогда не запускает модель, поэтому скрытых суждений нет.

→ Смотрите [**флагманские шаблоны**](../zh/FLAGSHIP-TEMPLATES.md) — hub'ы, которые нетехнический пользователь может импортировать и запустить сегодня (умный дом, операции кафе, семейный учебный hub, личный coding hub), каждый с явно показанными воротами управления и однокомандной демонстрацией. Хотите поделиться своим? [`templates/community/templates/`](../../templates/community/templates/).

## Основные идеи

- **Hub намеренно прост.** Он не запускает LLM и не владеет циклами агентов. Он маршрутизирует сообщения, распределяет задачи, сохраняет transcript и генерирует события. Решения остаются за участниками.
- **Люди — первоклассные участники.** Человек — это `Participant` так же, как и агент. Примитивы Hub для асинхронных / длительных операций применяются к обоим.
- **Один интерфейс, два варианта развёртывания.** Агенты реализуют один и тот же контракт `Participant` независимо от того, запускаются ли они в рамках процесса или через сеть. Локальные и удалённые агенты используют одинаковый реестр и один планировщик.
- **Подключаемое планирование.** Три стратегии маршрутизации задач из коробки: явное назначение, сопоставление возможностей и широковещательный захват.
- **Принеси свой LLM.** Небольшой базовый класс `LlmAgent` + нейтральный интерфейс `LlmProvider` позволяют снабдить агента Claude, GPT или любой другой моделью без изменения Hub.

## Статус

**Самостоятельно развёртываемый, файловый и управляемый для многоорганизационного использования.** Рабочее пространство — это директория на диске (`.gotong/`) — удалите директорию, и пространство исчезнет; скопируйте его — и вы передали комнату коллеге; перезапуски прозрачны. Помимо этого: хранилище учётных данных для каждой организации, межорганизационная федерация с контрактами доверия для каждого канала (allowlist возможностей · ворота классов данных · квота · отзыв), inbox'ы для подтверждения человека в контуре и журнал использования/стоимости. Hub по-прежнему никогда не запускает LLM — каждое решение остаётся за участниками.

Пакеты npm имеют пространство имён `@gotong/*`; Python SDK доступен как `gotong` на PyPI. Лицензия: [MIT](../../LICENSE).

### Добавление агента — два пути

|  | Управляемый хостом (без кода) | Внешний SDK (ваш код) |
|---|---|---|
| **Вы делаете** | Вставьте / загрузите YAML manifest в admin UI | Напишите `AgentParticipant.handleTask`, вызовите `connect(url, agents)` |
| **Где запускается** | Внутри процесса Hub (LocalAgentPool) | Где угодно в сети |
| **Что умеет** | LLM-задачи через провайдеров Anthropic / OpenAI / Mock | Всё — LLM, скрейперы, приватные данные, ML-модели, скрипты |
| **API-ключ хранится** | Зашифрован в `.gotong/secrets.enc.json` (для агента или по умолчанию для workspace) | Где угодно, где ваш код его читает |
| **При перезапуске** | Автоматически перезапускается `LocalAgentPool` | Ваш код переподключается (SDK имеет встроенный авто-повтор) |
| **Лучше для** | Конечные пользователи • стандартные роли • шаблоны в один клик | Разработчики • частная логика • кросс-языковые workers |

## Быстрый старт

### Нетехнический пользователь? Двойной клик, без Node/Docker

Путь, которому нужен **ни терминал, ни Node, ни Docker** на машине, которая его запускает. Мейнтейнер однажды собирает самодостаточный портативный пакет:

```bash
node scripts/build-portable.mjs        # → dist-portable/Gotong-macos-arm64/
```

Затем отдайте всю папку `Gotong-macos-arm64/` кому угодно. Они **делают двойной клик на `Gotong.command`** → браузер открывает 5-минутный мастер настройки. Пакет поставляется со своей фиксированной средой выполнения Node + скомпилированный хост + реальный `node_modules` на диске (включая нативную привязку SQLite), так что он запускает **полный** хост с поддержкой identity на машине, где ничего не установлено. Данные хранятся в `~/.gotong` (вне папки), поэтому замена пакета никогда не теряет данные.

Собирается по запросу, пока не является зафиксированной/опубликованной загрузкой (это план после версии 1.0) — пока что «скачать и запустить» означает *собрать папку один раз, поделиться папкой*. В этот раз macOS arm64. Полное описание: [`docs/zh/PORTABLE-BUNDLE.md`](../zh/PORTABLE-BUNDLE.md).

### Запустите за 30 секунд — выберите один вариант

```bash
# A. Docker (рекомендуется — не нужна настройка Node, работает на macOS / Windows / Linux)
docker compose up
# → http://127.0.0.1:3000  + admin URL напечатан в логах
# → состояние сохраняется в ./data

# B. Из исходников (клонированный репозиторий, доступен полный набор демонстраций)
pnpm install
pnpm build
pnpm host
```

Оба запускают один и тот же бинарный файл. Откройте напечатанный admin URL → сохраните токен → готово.

**Удобство при первом запуске (новое).** После загрузки хост печатает заметный баннер со следующим шагом, указывающий на мастер настройки loopback, и при первом локальном (loopback) запуске автоматически открывает браузер:

```text
┌─ 下一步 / Next step ──────────────────────────

  打开浏览器完成 5 分钟设置 (设置向导,无需 token):
  Open your browser to finish the 5-minute setup:

      →  http://127.0.0.1:3000

  设置向导在本机回环 (loopback) 上运行。
  The setup wizard runs on loopback only.
└───────────────────────────────────────────────
  (已自动打开浏览器 / browser opened — GOTONG_OPEN_BROWSER=0 关闭)
```

`GOTONG_OPEN_BROWSER` управляет автоматическим открытием: не задано = `auto` (только первый локальный запуск), `1`/`always` = каждый запуск, `0`/`never` = выключено. Также принудительно отключается всякий раз, когда хост открыт в сеть — headless-сервер никогда не открывает браузер, и мастер настройки там всё равно недоступен (этот путь использует файл admin-токена). Сам баннер всегда печатается.

> 💡 **Дистрибуция.** На данном этапе нет `npm publish` — Docker (A) и исходники (B) — это два поддерживаемых пути установки. Более ранний план «в очереди на v2.1» для npm был **исключён из области**; выбор реестра (npm / JSR / только исходники) — открытое решение, отслеживаемое в [RELEASE-CHECKLIST](.github/RELEASE-CHECKLIST.md). Предварительно собранные одиночные бинарные файлы для macOS / Windows — запланированный, но не блокирующий пункт — Docker уже покрывает кросс-платформенный случай «кликни и запусти».

CLI флаги (из собранного репозитория):

```bash
pnpm exec gotong-host --help       # полная справка по переменным окружения
pnpm exec gotong-host --version    # текущая версия хоста
```

После запуска следуйте [`docs/OVERVIEW.md`](../OVERVIEW.md) для прохождения «что делать дальше».

**Не запускается?** Запустите предварительную проверку перед загрузкой — она проверяет точные переменные окружения `GOTONG_*`, которые читает хост (версия Node, порты, доступные для привязки, директория данных доступна для записи, мастер-ключ) и печатает для каждой проверки ✓ / ⚠ / ✖ с однострочным исправлением:

```bash
pnpm exec gotong doctor          # только отчёт
pnpm exec gotong doctor --fix    # также автоматически создаёт отсутствующую директорию данных (единственное безопасное, обратимое исправление)
```

И если загрузка *всё же* завершается неудачно, хост превращает распространённые, исправимые сбои (порт уже используется, нет разрешения на привязку порта, отсутствующий/недействительный мастер-ключ, директория данных недоступна для записи, диск заполнен) в однострочное человекочитаемое сообщение, называющее, какую переменную `GOTONG_*` изменить — а не трассировку стека. Смотрите раздел устранения неполадок в [`docs/zh/GO-LIVE.md`](../zh/GO-LIVE.md) §十一.

**Проверьте, что ключевой зонд работает (реальный ключ не нужен).** Самая распространённая ловушка при первом запуске — вставленный LLM-ключ, который молча не работает. Мастер настройки перехватывает это с помощью пути спасения в один клик «去补 key»; эта команда проходит тот же зонд от начала до конца, чтобы вы знали, что путь спасения подключён до онбординга:

```bash
pnpm check:onboarding          # hermetic — доказывает, что плохой/пустой ключ → "добавьте ключ", сетевая ошибка → "проверьте URL"
ANTHROPIC_API_KEY=… pnpm check:onboarding   # также совершает полный проход с РЕАЛЬНЫМ ключом по сети (опционально; пропускается без него)
```

По умолчанию hermetic (без сети, без расходов) и никогда не записывает ваш ключ в лог. Код выхода 0 = каждая выполненная проверка прошла. Опциональная проверка с реальным ключом отражает контракт переменных окружения live gate (`OPENAI_API_KEY` + `OPENAI_BASE_URL=https://api.deepseek.com` + `GOTONG_LIVE_OPENAI_MODEL=deepseek-chat` для пути DeepSeek).

### Развёртывание на облачном сервере (VPS)

Есть свежий Ubuntu/Debian? Разместите checkout на нём (`git clone` с вашим ключом, или `scp` — репозиторий приватный, поэтому публичного pull нет), затем настройте systemd-сервис одной командой:

```bash
# изнутри checkout, на VPS
sudo bash deploy/cloud-quickstart.sh        # установить Node+pnpm → собрать → пользователь+unit
#   предварительный просмотр, ничего не меняет:  bash deploy/cloud-quickstart.sh --dry-run
```

Он устанавливает Node + pnpm, собирает, создаёт пользователя сервиса `gotong` и директорию данных, добавляет `/etc/gotong.env` (из [`deploy/.env.cloud`](../../deploy/.env.cloud)) и устанавливает systemd unit, отражающий [`docs/zh/DEPLOY.md`](../zh/DEPLOY.md) §C.4. **Останавливается на один шаг раньше запуска** — env-файл поставляется с пустыми значениями для домена / мастер-ключа / host-allowlist, и открытие ненастроенного бокса небезопасно. Он печатает безопасный финальный этап: заполните env, запустите [`scripts/cloud-harden.sh`](../../scripts/cloud-harden.sh) (проверка периметра), поставьте Caddy + firewall перед ним, затем `systemctl enable --now gotong`.

> **Кнопки «развернуть в один клик» в браузере нет**, пока репозиторий приватный (для них нужен публичный репозиторий или учётная запись провайдера, предварительно связанная с вашим git). Этот bootstrap, который можно вставить и выполнить, является реальным, тестируемым эквивалентом. Полное руководство — топология, риски открытия IP, онбординг членов IM: [`docs/zh/GO-LIVE.md`](../zh/GO-LIVE.md).

### 个人模式 (新, v4 Phase 7) — 一个人用 AI 干活, 0 配置

如果你就一个人, 想把 Gotong 当成"我的 AI 桌面"用 (不是给团队开 hub),
直接 `docker compose up` 就行 — host 第一次启动检测到只有你一个用户,
**自动进入个人模式**:

```bash
docker compose up
# → http://127.0.0.1:3000/admin?token=<打印出来>
# → 首屏顶部不显示 "owner" 角色 chip (个人用户不需要看见组织角色)
# → 副标题写"我的 AI 桌面"(不是"管理员控制台")
# → 设置 tab 出现 [升级到团队模式] 按钮 — 哪天想拉人就点一下
```

个人模式与团队模式的差别就两点:
- 主页副标题文案不同 / role chip 隐藏
- 设置里多个升级按钮

**所有 admin tab 都还在**(用户管理 / peer / 配额 / audit 全可见),
但你不会被这些概念占满屏幕。需要时再用。

`GOTONG_MODE=team` 可以强制 pin 团队模式(即使只有一个用户);
`GOTONG_MODE=personal` 反过来——多用户时也强制 pin 个人模式(罕见,
通常给 dev / 测试场景)。

升级到团队后, 自动出现"邀请用户"流程, 跟着导出 admin URL 给团队成员;
路径见下一节 5-min personal growth workflow 或 [`docs/zh/OVERVIEW.md`](../zh/OVERVIEW.md)。

### 5-минутный workflow личного роста (новое)

Первый готовый к работе опыт из коробки. 7 коучей (интервью + тело / психика / цели / ресурсы / отношения + комплексный планировщик) проходят один цикл → 12-недельный план на стене сохраняется на диск в формате markdown. По умолчанию LLM — **DeepSeek** (доступен в Китае, недорогой).

```text
1. Установите хост (Docker или исходники, см. выше)
2. Откройте напечатанный admin URL → войдите в admin
3. Получите API-ключ DeepSeek: https://platform.deepseek.com (новые пользователи получают 10 юаней кредита, этого достаточно для десятков запусков)
4. Admin → вкладка Workflows → нажмите [Импортировать команду (bundle)] → нажмите [🎁 Использовать встроенный шаблон: Personal Growth]
   → вставьте ключ DeepSeek → [Импортировать]
   (7 агентов создаются одним кликом, workflow регистрируется автоматически)
5. На карточке workflow нажмите [Начать] → появится 4-секционная форма (текущее положение / желания / блокировки / что хотите прояснить в этот раз)
6. Отправьте → подождите ~3,5 минуты (7 вызовов DeepSeek API)
7. Вкладка Workflows → прокрутите вниз → панель "Отчёт о росте" → нажмите [Скачать]
   или: <space>/services/artifact/file/agent/growth-synthesist/reports/<caseId>/<date>.md
```

Отчёт содержит: профиль + пять аналитических измерений (тело/психика/цели/ресурсы/отношения) + одна фраза пути развития + **12-недельный план** (основная + дополнительная линии, что делать каждую неделю) + **5 решений компромиссов** + запасной план «что делать, если не получается» + «5 вопросов-семян, на которые стоит ответить при следующем запуске workflow» (для следующего раза).

> 🙏 **О конфиденциальности / данных**: ваши 4 раздела самоописания отправляются DeepSeek (серверы в материковом Китае) для инференса. После завершения workflow все результаты сохраняются в директории `.gotong-*/services/` на вашем компьютере, никакие данные не загружаются в облако. Каждый коуч разработан как сопровождающий с границами — телесный коуч при наличии красных флагов (постоянная боль в груди / необъяснимое кровотечение и т.д.) направит к врачу; психологический коуч при сигналах риска предоставит 24-часовую линию кризисной помощи (национальная 400-161-9995 / Малайзия Befrienders 03-7956 8144). **Это не замена врачу / психологу-консультанту / финансовому советнику / семейному терапевту.**

Хотите заменить на Anthropic Claude или OpenAI? Отредактируйте `templates/teams/personal-growth-team.yaml`, измените `provider` / `baseURL` / `model` каждого агента — системные подсказки не зависят от поставщика.

### Логирование

Структурированное логирование **включено по умолчанию** — строка JSON на событие, когда stdout перенаправлен (для `jq` / Loki / ELK / Datadog), красиво отформатировано, когда stdout — терминал. Три переменных окружения управляют им:

```bash
GOTONG_LOG_LEVEL=info       # silent | trace | debug | info (по умолчанию) | warn | error | fatal
GOTONG_LOG_FORMAT=json      # json | pretty (по умолчанию: автоматически по TTY)
GOTONG_LOG_DISABLED=1       # аварийный выключатель полного отключения
```

Фильтруйте по компоненту с помощью `jq`, получив JSON-вывод:

```bash
pnpm host 2>&1 | jq 'select(.comp == "local-agents")'
```

## Распределённый — агенты подключаются из другого процесса / машины

Хост-процесс (Hub):

```ts
import { Hub } from '@gotong/core'

const hub = new Hub()

```

Worker-процесс (любой агент, где угодно):

```ts
import { AgentParticipant, connect } from '@gotong/sdk-node'

class MyAgent extends AgentParticipant {
  constructor() { super({ id: 'a1', capabilities: ['draft'] }) }
  protected async handleTask(task) { return { text: '…' } }
}

await connect({ url: 'ws://hub.example.com:4000', agents: [new MyAgent()] })
```

Вызовы `dispatch(...)` Hub'а достигают удалённого агента так же, как и локального. Смотрите [docs/PROTOCOL.md](../PROTOCOL.md) для формата wire и [examples/remote-agent](../../examples/remote-agent) для запускаемой двухпроцессной демонстрации.

## Агенты с поддержкой LLM

Hub не вызывает LLM. `LlmAgent` вызывает — это тонкий базовый класс, который связывает Task с `LlmProvider` и превращает ответ в `TaskResult`. Смена поставщика — изменение одной строки.

```ts
import { Hub } from '@gotong/core'
import { LlmAgent } from '@gotong/llm'
import { AnthropicProvider } from '@gotong/llm-anthropic'
import { OpenAIProvider } from '@gotong/llm-openai'

const hub = new Hub()
await hub.start()

// Claude пишет черновики
hub.register(new LlmAgent({
  id: 'writer',
  capabilities: ['draft'],
  provider: new AnthropicProvider(),        // читает ANTHROPIC_API_KEY
  system: 'You write one terse sentence.',
}))

// GPT их рецензирует
hub.register(new LlmAgent({
  id: 'reviewer',
  capabilities: ['review'],
  provider: new OpenAIProvider(),            // читает OPENAI_API_KEY
  system: 'You return one revision suggestion.',
}))

const draft = await hub.dispatch({
  from: 'system',
  strategy: { kind: 'capability', capabilities: ['draft'] },
  payload: { topic: 'distributed agents' },
})
```

Переопределите `buildRequest(task)` для настройки сборки подсказки (полученный контекст, few-shot примеры) или `parseResponse(response, task)` для постобработки (извлечение JSON, повторная подсказка для валидации). Переопределите `handleTask(task)` для полного контроля — многошаговые рассуждения, повторы, структурированные выходные данные. Смотрите [`packages/llm`](../../packages/llm/src/agent.ts) и две демонстрации в [`examples/llm-mock`](../../examples/llm-mock) и [`examples/llm-real`](../../examples/llm-real).

## Open Space — администраторы, workers и агенты в одной комнате (v2.0)

Привяжите hub к директории `.gotong/`; идентификация администратора, учётные записи workers и контролируемые допуски агентов — всё там. Веб-интерфейс разделяется на два вида (`/` worker, `/admin` администратор). Перезапуски Hub прозрачны — куки всё ещё работают, администраторы остаются администраторами, transcript растёт, а не перезапускается.

```ts
import { Hub, Space } from '@gotong/core'
import { serveWebSocket } from '@gotong/transport-ws'
import { serveWeb } from '@gotong/web'

const { space, adminToken } = await Space.openOrInit('.gotong', {
  name: 'my-space',
  adminDisplayName: 'Operator',
  config: { gating: 'admin-approval' },
})
console.log(`Admin URL once: http://localhost:3000/admin?token=${adminToken}`)

const hub = new Hub({ space })
await hub.start()

await serveWebSocket(hub, { port: 4000, gating: (await space.config()).gating })
await serveWeb(hub, { port: 3000 })
// admin = /admin?token=<TOKEN>   |   worker = /
```

- **Администратор** входит один раз с токеном, затем управляет комнатой: подтверждает / отклоняет ожидающие допуски агентов, отправляет задачи через любую из трёх стратегий, видит все задачи на фильтруемой панели с кнопкой **Повторить** на неудавшихся строках, пишет оценки, привязанные к конкретным задачам.
- **Worker** выбирает никнейм + возможности на `/`, становится `HumanParticipant`. Строка `workers.json` + HttpOnly-куки запоминают их между перезагрузками и перезапусками.
- **Агент** подключается к порту WebSocket; при `gating: 'admin-approval'` они ожидают в очереди, пока администратор не примет меры.

Полная запускаемая демонстрация в [`examples/open-space`](../../examples/open-space). `pnpm demo:open-space` запускает хост + агент в одном терминале, затем укажите браузер на два URL, которые он печатает.

## Hub Services — память агента, артефакты, хранилища данных (v2.2)

Агент может объявить, какое состояние он хочет, чтобы хост хранил от его имени. Сегодня поставляются три первопартийных «сервиса»; сантехника — плагинная с первого дня, поэтому добавление четвёртого — отдельный npm-пакет.

```yaml
# templates/agents/industry-coach-with-memory.yaml
schema: gotong.agent/v1
agent:
  id: industry-coach
  capabilities: [intake]
  provider: anthropic
  model: claude-opus-4-7
  system: |
    Use memory.recall before answering; artifact.write the report
    afterwards; cases.sql for structured industry comparisons.
  uses:
    - { type: memory,    impl: file,   config: { kinds: [episodic, semantic] } }
    - { type: artifact,  impl: file,   config: { name: industry-reports } }
    - { type: datastore, impl: sqlite, config: { name: cases, schema: "..." } }
```

При запуске хост разрешает каждую запись `uses:` в типизированный дескриптор, который агент читает из `ctx.memory`, `ctx.artifact`, `ctx.datastore.<name>`. Изоляция на основе владельца — по умолчанию — два агента, запрашивающие `memory:file`, получают два разных хранилища. Расположение данных находится в `<space>/services/`:

```
<space>/services/
├─ plugins.json                    # какие плагины загружать (автоматически инициализируется)
├─ memory/file/agent/<agentId>/    # одна директория на (плагин, владелец)
├─ artifact/file/agent/<agentId>/
└─ datastore/sqlite/agent/<agentId>/<name>.sqlite
```

Мягкое удаление — это клик на вкладке admin «服务 / Services»; данные перемещаются в `.trash/` для каждого плагина, живут 30 дней, затем фоновый sweeper жёстко удаляет их. Восстановление — один POST до тех пор. Полный дизайн в [`docs/services-rfc.md`](../services-rfc.md).

| Пакет | Что предоставляет |
|---|---|
| `@gotong/services-sdk` | Контракт `ServicePlugin`, реестр, загрузчик. Шов, который реализуют авторы плагинов. |
| `@gotong/service-memory-file` | Первопартийный `memory:file` — episodic / semantic / working как JSONL. |
| `@gotong/service-artifact-file` | Первопартийный `artifact:file` — директории файлов для каждого владельца с защитой MIME + размера. |
| `@gotong/service-datastore-sqlite` | Первопартийный `datastore:sqlite` — KV + raw SQL на одном `.sqlite` для каждого объявленного имени. |

### Написание своего плагина

```ts
// my-plugin/src/index.ts
import type { ServicePlugin } from '@gotong/services-sdk'

class MyPlugin implements ServicePlugin {
  readonly type = 'memory'
  readonly impl = 'redis'
  readonly version = '0.1.0'

  async init(ctx) { /* открыть пул redis */ }
  async validateConfig(raw) { /* разобрать + отклонить плохие формы */ }
  async attach(owner, config) { /* вернуть MemoryHandle */ }
  async detach(owner) { /* закрыть кеш для каждого владельца */ }
  async softDelete(owner) { /* вернуть TrashRef; хост сохраняет его */ }
  async restore(ref) { /* выбрасывает TrashRestoreConflictError при конфликте */ }
  async hardDelete(ref) { /* необратимо */ }
  async describe(owner) { /* снимок admin UI — sizeBytes, preview */ }
  async shutdown() { /* drain + close */ }
}

export default () => new MyPlugin()
```

Добавьте имя пакета в `<space>/services/plugins.json` и перезапустите хост — `loadPlugins` динамически импортирует запись, вызывает `init` и плагин доступен для каждого `uses:` yaml агента. Сбои загрузки плагина не фатальны: плохой плагин отображается в журнале загрузки, но не приводит к сбою хоста.

> **Замечание по развёртыванию**: хост разрешает пакеты плагинов из своего `node_modules/`, поэтому сторонние плагины нужно установить там, где их может видеть хост — `pnpm add my-org/gotong-redis-memory` в workspace хоста или зависимость `package.json` в образе развёртывания. Одного размещения имени пакета в `plugins.json` недостаточно, если сам пакет не находится на диске.
