# Флагманские шаблоны — хабы, которые обычный человек может импортировать и использовать

<!-- doc-version: 1.0 -->
> **Версия документа 1.0** · Перевод на русский язык · Обновлено: 2026-06-27 · Авторитетный источник: [English](../FLAGSHIP-TEMPLATES.md). При расхождении перевода с английской версией приоритет имеет английская версия.

> Это **одобренный** список шаблонов. «Флагманский» не означает «лучший» — это значит «мы ручаемся за него»: каждый из них поставляется с **детерминированным демо** (одна команда, без ключа, проверяет своё собственное поведение), каждый выставляет свою **позицию управления** (что может трогать, что нет, где стоит человеческий шлюз) на всеобщее обозрение, и каждый **поддерживается**.
>
> Хотите увидеть все шаблоны (включая уровень сообщества)? Раздел «Рабочие процессы → Галерея шаблонов» в административном интерфейсе. Хотите отправить свой: [`templates/community/templates/`](../templates/community/templates/). Критерии отбора для этого списка написаны в [`GOVERNANCE.md`](../GOVERNANCE.md).

## Почему именно эти

Отличие Gotong — не в «умении вызывать AI» — это есть везде. Оно в том, что **вы осмеливаетесь направить AI на свой дом, семью, деньги**, потому что границы реальны и они ваши:

- **Человек стоит у шлюза критических действий.** Обратимые (выключить свет) просто выполняются; необратимые (запереть дверь, потратить деньги, отправить данные ребёнка) приостанавливаются и ждут, пока человек не подтвердит в почтовом ящике — рабочий процесс **не может обойти** этот шлюз.
- **Ключи и данные на вашем диске.** Учётные данные зашифрованы в вашем каталоге `.gotong/`. Федерация с другим хабом разделяет **возможность**, а не ваш vault.
- **Нет решений «чёрного ящика».** Каждая диспетчеризация и результат — это читаемый, доступный только для чтения транскрипт. Фреймворк никогда не запускает модель; нет скрытых суждений.

Каждый из приведённых ниже шаблонов — это три принципа, **воплощённые в одной конкретной вещи**.

## Дом и семья

### ⭐ smart-home-hub — Умный дом (Xiaomi через Home Assistant)

**Для кого / что.** Домашний управляющий управляет вашими устройствами Xiaomi (или любыми интегрированными с HA) через Home Assistant, выполняя «вечернюю рутину».

**Что может трогать.** Выключить свет в общих зонах, переключить кондиционер в спальне в спящий режим — это **обратимо**, просто выполняется.

**Где стоит шлюз человека (позиция управления).** Запирание входной двери и включение охраны — это **необратимые физические/охранные** действия — рабочий процесс, достигая этого шага, **приостанавливается** и ждёт, пока жилец нажмёт «подтвердить» в почтовом ящике `/me` перед выполнением. Отказ → этот шаг пропускается шлюзом `when:` → **дверь остаётся незапертой** (fail-closed, блокирует следующее действие, без побочных эффектов). Вот как выглядит «обратимое выполняется напрямую, необратимое требует подтверждения человека» в домашних условиях.

**Разделение шаблонов/фреймворка.** MCP-подключение устройств в шаблоне — это заполнители `${HA_MCP_SSE_URL}` / `${HA_TOKEN}` — какой Home Assistant вы подключаете и какой токен используете, это конфигурация runtime, заполняемая после импорта. Рабочий процесс называет только возможности (`home.apply-scene` / `home.secure`), никогда конкретное устройство. Смените устройства, смените дом — рабочий процесс не изменится ни на слово. В этом шаблоне **нет слота KB** (состояние устройств — живой HA, отдельная база знаний не нужна).

- Запустить: `pnpm demo:smart-home-hub` (два сценария: одобрить → дверь запирается; отказать → дверь остаётся незапертой)
- Шаблон: [`examples/smart-home-hub/template/smart-home-hub.template.yaml`](../examples/smart-home-hub/template/smart-home-hub.template.yaml)
- Подключение реального Home Assistant: см. [README](../examples/smart-home-hub/README.md)

### ⭐ family-learning-hub — Семейное обучение (родители открывают AI для детей)

**Для кого / что.** Родитель платит за AI-подписку, ребёнок учится на **отдельном** хабе; хаб ребёнка вызывает подписку родителя через авторизацию, а AI-репетитор (воссоздание `/teach` Matt Pocock: сначала установить миссию, один маленький шаг, знания перед навыком, цитировать первоисточник) направляет исследование ребёнка. Это **наиболее готовый к производству** в списке (настоящая ws-федерация + IM-наблюдение + настоящий DeepSeek — всё проверено).

**Что может трогать.** В пределах тем из белого списка репетитор обучает напрямую; **основная копия** учебных записей находится на хабе ребёнка.

**Где стоит шлюз человека (позиция управления) — четыре шлюза.**

1. **Белый список тем + самооценка контента** → темы вне белого списка и контент, который репетитор самостоятельно пометил как `flagged`, **приостанавливаются для родительского одобрения**.
2. **Шлюз классификации данных**: данные ребёнка помечены `child-learning` и не могут быть отправлены третьей стороне, не авторизованной для этого класса данных (fail-closed).
3. **Юрисдикция**: родитель держит подписку (экономическое узкое место) + договор доверия на каждую ссылку федерации + форк транскрипта повсюду (родитель получает копию для наблюдения).
4. **Учётные данные / данные остаются дома**: два суверенных хаба, данные ребёнка отправляют копию родителю со стороны ребёнка, но подписка и vault не пересекаются.

**Разделение шаблонов/фреймворка.** Межорганизационная ссылка (какой пир-ребёнок, какие возможности разрешены исходящим, политика одобрения, `allowedDataClasses`) — это **конфигурация пира runtime**, ни в шаблоне, ни в рабочем процессе. Два шаблона: на стороне родителя `family-tutor` (с репетитором + рабочим процессом белого списка/одобрения), на стороне ребёнка `child-desk` (без подписки + основная копия учебных записей).

- Запустить: `pnpm demo:family-learning-hub` (шесть сценариев, включая вне-белого-списка→родитель одобряет / родитель отказывает→урок не проведён)
- Шаблоны: [`family-tutor`](../examples/family-learning-hub/template/family-tutor.template.yaml) · [`child-desk`](../examples/family-learning-hub/template/child-desk.template.yaml)
- Реальное развёртывание (две суверенные машины): [`docs/zh/FAMILY-LEARNING-GO-LIVE.md`](zh/FAMILY-LEARNING-GO-LIVE.md) · Дизайн: [`FAMILY-LEARNING-HUB-DESIGN.md`](zh/FAMILY-LEARNING-HUB-DESIGN.md)

## Личная продуктивность

### personal-coding-hub — Личное программирование (разделение труда Claude Code + Codex)

**Для кого / что.** «Модель» маршрутизации анализирует задачу + учитывает вашу договорённость и решает, диспетчировать ли работу Claude Code или Codex; два агента программирования разделяют один рабочий каталог и взаимодействуют через `AGENTS.md` (спецификация) + `PROGRESS.md` (эстафетная палочка). Есть также **состязательная консультация**: когда возникает проблема, несколько агентов читают код вместе, сначала диагностируют вслепую, затем перекрёстно проверяют и голосуют, чтобы сойтись на реальной первопричине.

**Где стоит шлюз человека (позиция управления).** Опасные команды (`rm -rf`, `git push --force`, `sudo`, `curl | sh` …) приостанавливаются **перед** выполнением для вашего одобрения; отказ → fail-closed, команда так и не выполнена. Разделение труда — **ваш выбор**: назовите его ситуативно («дай это codex») или измените общий уровень разделения на простом языке (в стиле OpenClaw, записывается обратно в `routing-policy.json`).

**Разделение шаблонов/фреймворка.** Шаблон несёт 1 агента-наставника (`coding-mentor`, DeepSeek + встроенный mcp-obsidian) + 1 адресуемый слот KB (библиотека методологии, указатель `presetData`). Два CLI-агента программирования **подключаются во время выполнения** (CliParticipant не попадает в реестр управляемых агентов); **содержимое** знаний живёт вне шаблона.

- Запустить: `pnpm demo:personal-coding-hub` (10 сценариев: разделение труда / явное назначение / переразделение на простом языке / шлюз безопасности)
- Консультация: `pnpm demo:personal-coding-hub:consult`
- Шаблон: [`examples/personal-coding-hub/template/personal-coding-hub.template.yaml`](../examples/personal-coding-hub/template/personal-coding-hub.template.yaml)

### codex-deepseek-hub — Программирование (Codex + DeepSeek TUI)

**Дочерний** personal-coding-hub: другой набор моделей — Codex (быстрый исполнитель) + DeepSeek TUI (ведущий с рассуждением). Та же маршрутизация + переразделение на простом языке + явное назначение + шлюз безопасности, самостоятельный и не затрагивающий personal-coding-hub.

- Запустить: `pnpm demo:codex-deepseek-hub`
- Шаблон: [`examples/codex-deepseek-hub/template/codex-deepseek-hub.template.yaml`](../examples/codex-deepseek-hub/template/codex-deepseek-hub.template.yaml)

### personal-research-hub — Личные исследования / хаб знаний

**Для кого / что.** Библиотекарь **компилирует** ваш сырой исходный материал во взаимосвязанную Obsidian-вики (LLM-как-компилятор), затем позволяет «спрашивать вашу вики». Три управляемых LLM-агента (библиотекарь / компилятор / исследователь) переходят командой.

**Позиция управления.** Компиляция — это **только чтение**, превращение сырого в заметки + обратные ссылки; ответы цитируют источники и архивируются в `wiki/answers/`.

- Запустить: `pnpm demo:personal-research-hub`
- Шаблон: [`examples/personal-research-hub/template/personal-research-hub.template.yaml`](../examples/personal-research-hub/template/personal-research-hub.template.yaml)

### battle-monk-training — Личный рост (тело / разум / знания, три столпа)

**Для кого / что.** Наставник диспетчеризует сегодняшнее испытание к трём столпам (тело / разум / знания), каждый продвигается к следующему рангу на основе рангов, уже тренированных в вашей записи, с непрерывностью как основным принципом дизайна — Obsidian KB **хранит ваше состояние** (не справочный материал). Холодный grimdark-монастырский стиль (оригинальный фанатский трибьют, ориентированный на пользователей в стиле Warhammer 40k).

**Позиция управления / граница безопасности.** **Записывает только ваш собственный протокол роста**; это личные данные, **не медицинские / психологические советы** — не используйте это как единственную основу ни для чего.

- Запустить: `pnpm demo:battle-monk-training`
- Шаблон: [`examples/battle-monk-training/template/battle-monk-training.template.yaml`](../examples/battle-monk-training/template/battle-monk-training.template.yaml)

## Организации и межорганизационное

### cafe-ops — Операции магазина (магазин пузырькового чая / кофейни)

**Для кого / что.** Формальные процессы небольшого магазина: адаптация новых сотрудников (изучение СОП позиции, самообслуживание участников), расписание (подтверждение менеджера), оплата сверхурочных (одобрение менеджера). Первый шаблон с непустым `workflows[]` — ценность организации в формальном процессе.

**Где стоит шлюз человека (позиция управления).** Оплата сверхурочных: **ассистент только предлагает сумму, менеджер решает деньги**: ассистент вычисляет множитель по типу дня (будний день 1,5 / выходной день 2 / официальный праздник 3), но рабочий процесс, достигая шага одобрения, приостанавливается и вступает в силу только после того, как менеджер одобряет в почтовом ящике. **Деньги вычисляются детерминированно, а не LLM; человек решает.**

- Запустить: `pnpm demo:cafe-ops` (включает двухэтапное возобновление HITL сверхурочных)
- Шаблон: [`examples/cafe-ops/template/cafe-ops.template.yaml`](../examples/cafe-ops/template/cafe-ops.template.yaml)

### warband-club — Клуб любителей (общий архив)

**Для кого / что.** **Сторона сотрудничества** сообщества по интересам / военной группы (в отличие от управленческой стороны cafe-ops): общий архив, который вся группа читает и пишет — схема покраски / боевой отчёт, который вы отправляете, другие могут найти; ответ, который вы получаете, может прийти от более раннего вклада кого-то другого = сотрудничество.

**Позиция управления.** Общий архив может читать и писать каждый; важные решения (сбор) проходят через подтверждение `human:` лидера. Разделяется внутри одного хаба, без федерации.

- Запустить: `pnpm demo:warband-club`
- Шаблон: [`examples/warband-club/template/warband-club.template.yaml`](../examples/warband-club/template/warband-club.template.yaml)

### tea-supply-link — Межорганизационная цепочка поставок (чайный магазин ↔ поставщик)

**Для кого / что.** Первый **межорганизационный** шаблон: рабочий процесс пополнения запасов чайного магазина оркестрирует один шаг на **хаб поставщика**.

**Где стоит шлюз человека (позиция управления).** Межорганизационный шаг заказа проходит через **шлюз утверждения исходящих** (прозрачный для рабочего процесса, так что в рабочем процессе **нет** шага `human:`) — только после того, как менеджер одобряет, он пересекает границу, поставщик рассчитывает цену построчно по каталогу + текущим запасам, и квитанция возвращается для локального хранения. Поставщик считает деньги, человек решает отправить.

**Разделение шаблонов/фреймворка (обучающий момент).** Межорганизационная ссылка (какой пир является поставщиком, какие возможности разрешены исходящим, политика одобрения) — это **конфигурация пира runtime**, ни в шаблоне, ни в рабочем процессе — шаг `place` только записывает возможность `supplier.confirm-order`, никогда не называя пира.

- Запустить: `pnpm demo:tea-supply-link`
- Шаблон (сторона магазина): [`examples/tea-supply-link/template/tea-shop.template.yaml`](../examples/tea-supply-link/template/tea-shop.template.yaml)
- Двухмашинный операционный сборник: [`docs/zh/FEDERATION-RUNBOOK.md`](FEDERATION-RUNBOOK.md)

### tea-chain-hq — Штаб сети (штаб → магазины франшизы)

**Для кого / что.** **Зеркало, обратное направление** tea-supply-link: тот идёт вверх (магазин→поставщик), этот идёт вниз (штаб→магазин франшизы). В трёхуровневой цепочке `штаб → магазин → поставщик` магазин находится посередине.

**Где стоит шлюз человека (позиция управления).** Межорганизационный шаг развёртывания директивы о перепрайсинге проходит через шлюз утверждения исходящих — только после того, как региональный менеджер одобряет, он пересекает границу, магазин детерминированно применяет перепрайсинг по своему меню, и квитанция возвращается. **Магазин — суверенная организация, а не подчинённый объект.**

- Запустить: `pnpm demo:tea-chain-hq`
- Шаблон (сторона штаба): [`examples/tea-chain-hq/template/chain-hq.template.yaml`](../examples/tea-chain-hq/template/chain-hq.template.yaml)

## Запустите любой одной командой (детерминированно, без ключа)

У каждого флагмана есть **детерминированное демо**: запускает весь процесс с детерминированными заменителями, проверяя собственное поведение, без API-ключа, без реального устройства / реального аккаунта. Это проверяемая половина «мы ручаемся за него» — одна команда доказывает, что он действительно работает:

```bash
pnpm demo:smart-home-hub          # дом: одобрить→дверь запирается / отказать→дверь остаётся незапертой
pnpm demo:family-learning-hub     # семья: вне-белого-списка→родитель одобряет / родитель отказывает→урок не проведён
pnpm demo:cafe-ops                # магазин: HITL сверхурочных, менеджер решает деньги
pnpm demo:personal-coding-hub     # программирование: разделение труда + шлюз безопасности
pnpm demo:personal-research-hub   # исследования: сырое → взаимосвязанная вики
pnpm demo:battle-monk-training    # рост: тело/разум/знания три столпа
pnpm demo:warband-club            # клуб: общий архив + подтверждение лидера
pnpm demo:tea-supply-link         # межорганизационное: пересечение границы требует одобрения человека
pnpm demo:tea-chain-hq            # сеть: развёртывание перепрайсинга требует одобрения человека
pnpm demo:codex-deepseek-hub      # программирование (Codex + DeepSeek)
```

Чтобы увидеть, как сам шаблон парсится (предварительный просмотр загрузки, тоже без ключа): замените любую из вышеперечисленных команд на `pnpm demo:<name>:template`.

## Реальное использование

Детерминированное демо доказывает, что логика работает; для реального использования флагмана выберите эти пути:

- **Установка в один клик**: нажмите одну в разделе «Рабочие процессы → Галерея шаблонов» административного интерфейса, и она установится в ваш хаб (см. [`docs/zh/TEMPLATE-GALLERY.md`](zh/TEMPLATE-GALLERY.md)).
- **Сравнение личного/организационного хаба + реальное внедрение DeepSeek/Obsidian**: [`docs/zh/HANDS-ON-HUBS.md`](zh/HANDS-ON-HUBS.md).
- **Запуск (три топологии)**: [`docs/zh/GO-LIVE.md`](zh/GO-LIVE.md).
- **Двухмашинный сборник межорганизационной федерации**: [`docs/zh/FEDERATION-RUNBOOK.md`](FEDERATION-RUNBOOK.md).
- **Развёртывание семейного обучения на двух суверенных машинах**: [`docs/zh/FAMILY-LEARNING-GO-LIVE.md`](zh/FAMILY-LEARNING-GO-LIVE.md).

## Хотите внести свой

Флагманов мало и они одобрены. Подавляющее большинство шаблонов должно быть **уровня сообщества** — планка «лицензионная чистота, парсится, ноль секретов в открытом тексте, есть происхождение», а не «мы ручаемся за ваш вкус». Процесс описан в [`templates/community/templates/README.md`](../templates/community/templates/README.md): скопируйте флагман → адаптируйте под свои нужды → объявите происхождение (`derivedFrom`) → `pnpm check:templates` локально → откройте PR.

Честное происхождение — валюта этого сообщества: `derivedFrom` направляет заслугу апстриму, а статическая таблица лидеров цитирования просто считает «сколько шаблонов произведено от вас». Продвижение с уровня сообщества до флагмана — это решение мейнтейнера на публичном issue — критерии описаны в [`GOVERNANCE.md`](../GOVERNANCE.md).
