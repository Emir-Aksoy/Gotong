## 1. 강점 매트릭스: 제품 × 차원 (Gotong는 마지막 행)

**차원 용어집**: 교차 조직 연합 = 서로 다른 주권 허브/조직의 에이전트가 협업 (자격증명은 각각 집에); 개인↔조직 연속성 = 개인 모드에서 팀, 그 다음 교차 조직으로 원활하게 확장되는 하나의 스택; 프레임워크가 LLM 미실행 = 프레임워크가 라우팅/계산만 하고 결코 참여자 대신 결정하지 않음 (허브는 멍청함).

**이 표를 읽는 방법**:
- **가장 오른쪽 세 열** (교차 조직 연합 / 개인↔조직 / 프레임워크 LLM 미실행)은 **Gotong만 ✅**이며, 나머지는 모두 ❌/⚠️ — 이것이 Gotong의 진정한 공백입니다.
- 상업 플랫폼 (상위 4행): 거버넌스/HITL에서 최고점이지만, **OSS·자체 호스팅·데이터 소유는 모두 ❌** (SaaS, 벤더가 신뢰 모델 보유; 일부는 VPC를 제공하지만 여전히 밑바닥은 임차).
- OSS 프레임워크 / 자체 호스팅 플랫폼 (중간 7행): OSS·자체 호스팅에서 최고점이지만, **거버넌스 ⚠️만, 교차 조직 모두 ❌** (한 조직 내에서만).
- 개인 에이전트 (OpenClaw/Hermes/Goose/Odysseus): 자체 호스팅에서 최고점이지만, **거버넌스·교차 조직·연속성 경로 모두 ❌** (단일 인원, 단일 기기).
- **Gotong만이 유일하게 모두 초록색 행** — 하지만 정확히 차원이 자체 입장에 따라 선택되었으므로, 아래의 약점 매트릭스를 함께 읽어야 합니다.

## 2. 약점 매트릭스: 솔직한 역방향 비교 (엔터프라이즈 구매자가 실제로 신경 쓰는 차원으로 교체하면 Gotong가 더 약함)

위의 표는 "홈 어드밴티지"에서 승리합니다. 엔터프라이즈 구매자가 실제로 묻는 차원으로 교체하면 격차가 즉시 역전됩니다:

| 차원 | 강한 쪽 (특정 제품) | Gotong |
|---|---|---|
| 고객 검증 / 규모 | Salesforce Agentforce (8000+ 고객), ServiceNow | ❌ 초기 단계 / 자체 사용 |
| 생태계 통합 (CRM/ITSM/Office/SAP) | Salesforce, ServiceNow, Microsoft | ❌ MCP를 통해 직접 연결 |
| 컴플라이언스 인증 (SOC2/ISO/HIPAA) | 모든 상업 플랫폼 | ❌ 인증 없음 |
| 기본 제공 / 노코드 성숙도 | n8n, Dify, Agentforce | ⚠️ 구성 필요 / 예제 우선 |
| 번들된 강력한 모델 + SLA + 상업 지원 | 모든 상업 플랫폼 | ❌ (그리고 설계상 모델 미실행; 연결하는 MiMo/DeepSeek/Claude에 의존) |
| 시각적 오케스트레이션 성숙도 | n8n, Flowise, Dify | ⚠️ 주로 선언적 YAML (읽기 전용 DAG 보기 추가됨) |

**네트워크 효과 약점 (별도 나열, 연합 클래스 제품에게 생사의 문제이기 때문)**: 연합의 가치는 피어 수에 따라 초선형으로 증가하는 반면, 콜드 스타트에서 피어 수 = 0입니다. 이것이 모든 "교차 조직" 제품의 죽음의 함정 — 아래 §4는 우리가 선택한 타깃 사용자가 **자체 피어를 가져와 이 함정을 피하는** 이유를 설명합니다.

## 3. 한 줄 결론

**Gotong는 "더 나은 Agentforce" 또는 "더 강력한 n8n"이 아닙니다; 아무도 점유하지 않는 교차로 셀에 있습니다**: 자체 호스팅 주권 + 교차 조직 연합 + 조직 수준 거버넌스/HITL + 개인-조직 연속성 경로 + 프레임워크가 LLM 미실행. 대가는 "성숙도 / 생태계 / 컴플라이언스 / 고객 검증"에서 초기 단계 제품 — 정확히 상업 플랫폼이 가장 강한 곳입니다.

## 4. 우리에게 가장 잘 맞는 사용자 — "필요가 있지만 오늘날 충족되지 않음"

두 매트릭스를 쌓아보면 **이미 서비스되는 셀들은 붐비고; 충족되지 않은 셀은 하나뿐입니다**:

> **거버넌스 / 후견 / 승인 + 데이터 주권 + 경계를 넘는 협업이 필요하지만, ① 엔터프라이즈 등급 플랫폼을 감당하거나 사용할 수 없고, ② OSS 프레임워크의 "단일 조직, 거버넌스 없음" 상한선에 의해서도 배제되는 소규모 조직들.**

이 셀은 오늘날 거의 진공 상태입니다 — 아무도 구축하고 싶지 않아서가 아니라, **두 개의 벽에 동시에 막혀 있기 때문**:

- **벽 A (가격/성숙도)**: 엔터프라이즈 플랫폼 (Agentforce/ServiceNow/Microsoft/Google)은 거버넌스와 HITL을 갖추고 있지만, 고가의 고터치 SaaS GTM이며 **단순히 가정집, 버블티 가게, 세 명의 법률 사무소에는 내려가서 팔지 않습니다**.
- **벽 B (아키텍처)**: OSS 프레임워크 (LangGraph/Dify/n8n)와 개인 에이전트 (OpenClaw/Goose)는 충분히 저렴하고 자체 호스팅이 가능하지만, **아키텍처적으로 교차 조직 연합, 조직 수준 거버넌스, 출스탄드 승인 게이트가 없습니다** — 아무리 저렴해도 그 능력을 키울 수 없습니다.

Gotong는 정확히 두 벽 사이의 틈에 서 있습니다: 엔터프라이즈 플랫폼의 "거버넌스 + HITL + 데이터 주권"을 가지면서도, OSS 프레임워크의 "자체 호스팅 + 저렴 + 자격증명이 내 기기에"를 갖추며, **거기에 더해 세 열을 독점합니다 (교차 조직 연합 / 개인↔조직 연속성 / 프레임워크 LLM 미실행)**.

### 4.1 두 가지 가장 예리한 교두보 (이미 예제를 통해 구축 완료)

| 교두보 | 누구 | 오늘날 충족되지 않는 이유 | 우리가 구축한 예제 |
|---|---|---|---|
| **A. 가정 / 교육** | 아이를 위해 AI를 여는 부모, 다구성원 가족 주권 AI, 부모 후견+승인 | 엔터프라이즈 플랫폼은 가족에게 팔지 않음; 개인 에이전트는 후견/구성원 간 주권/승인 게이트 없이 단일 인원/단일 기기 | `family-learning-hub` (두 주권 허브 + 출스탄드 승인 게이트 + 아동 데이터 데이터 클래스 잠금), `/teach` 튜터, 부모에게 트랜스크립트 포크 |
| **B. SMB 교차 조직** | 공급망 (가게↔공급업체), 프랜차이즈 체인 (본사↔매장), 멘토십/클럽/회사 간 프로젝트 | 엔터프라이즈 플랫폼은 소규모에 너무 무겁고 비쌈, 그리고 그들의 교차 조직 이야기는 여전히 벤더 종속; OSS 프레임워크는 단일 조직 | `tea-supply-link`, `tea-chain-hq`, `warband-club`, `cafe-ops` |

한 링 더 확장하면, 같은 셀은 **규제받는 소규모 팀의 연합**도 포함합니다: 법률 사무소 컨소시엄, 클리닉, 회사 간 RFP, 연구 협업 — 모두 "교차 조직 협업 + 데이터는 내 손에 + 감사/승인 필요"이며, 오늘날 동등하게 충족되지 않습니다.

### 4.2 DeepSeek가 "가격 벽"을 밀어 넘었습니다 — 정확히 사용자가 지목한 변수

사용자의 판단은 완전히 옳습니다: **이전에 가격/성능 근거가 없었던 제품이 나중에는 있을 수 있습니다.** 메커니즘을 풀어서 설명하면:

1. **이 셀은 역사적으로 이중 제약에 갇혀 있었습니다**: ① LLM이 너무 비쌈 + ② "주권 + 거버넌스 + 교차 조직 + 소비자 가격"을 채우는 제품이 없음. 가정은 감당할 수 없고, 버블티 가게는 마진이 얇으며, "모든 상호작용에서 LLM을 실행하고 라우팅/상담/하트비트를 위해 여러 에이전트를 살아있게 유지"하는 것이 **구형 모델 가격에서는 수지가 맞지 않았습니다** — 그래서 이런 종류의 자체 호스팅, 거버넌스 스타일 AI는 데모에 머물렀고, 실제로 라이브한 사람이 없었습니다.
2. **DeepSeek의 지난 두 달간의 새 API는 제약 ①을 제거합니다** (LLM 비용). **Gotong는 정확히 제약 ②를 채웁니다** (누락된 제품). 둘을 합치면 이 셀은 처음으로 "저렴함"과 "사용할 것"을 동시에 가지게 됩니다.
3. **핵심 비대칭 — 경쟁자도 저렴한 DeepSeek를 사용할 수 있지만, 저렴한 LLM이 이 셀에 도달하는 데 도움이 되지 않습니다**:
   - 엔터프라이즈 플랫폼: 저렴한 LLM이 고터치 엔터프라이즈 GTM을 바꾸지 않습니다; 그들은 토큰을 절약하기 위해 가정/소규모 가게에 내려가서 팔려 하지 **않을 것입니다**.
   - OSS 프레임워크: 저렴한 LLM이 교차 조직 연합/거버넌스/HITL을 **추가할 수 없습니다** — 가격 인하가 아키텍처의 부재를 채울 수 없습니다.
   - 개인 에이전트: 저렴한 LLM이 후견/교차 조직/승인을 **추가할 수 없습니다** — 설계상 단일 인원/단일 기기입니다.
   - → DeepSeek는 모두를 들어올리는 밀물이지만, **Gotong의 셀을 불균형적으로 여는 것입니다**: 그 셀은 "비용 ∧ 누락된 제품" 동시에 막혀 있었고, DeepSeek가 비용을 제거하면 **오직 Gotong만이 누락된 제품을 공급합니다**.
4. **그리고 저렴한 LLM은 다른 것보다 Gotong에 더 유익합니다** — 간과된 포인트: Gotong의 "프레임워크가 LLM 미실행, 하지만 참여자들이 실행" 설계는 자연스럽게 **많은 소규모 LLM 호출**을 만듭니다 (누구에게 디스패치할지 결정하는 라우팅 에이전트, 다중 에이전트 상담, 하트비트 능동적 깨우기, 세 기둥 성장 에이전트, 튜터+토픽 스크리닝+콘텐츠 조정…). 이 "많은 저렴한 LLM 참여자" 형태는 구형 모델 가격에서 **비용 부담**이었습니다 — 정확히 비엔터프라이즈 사용자를 배제한 것; DeepSeek가 단위 가격을 낮추면, Gotong의 가장 자연스러운 설계가 **가격/성능 스윗스팟**이 됩니다 — 그리고 정확히 이전에 가격이 짜냈던 사용자들에게 달콤합니다.

### 4.3 이 선택이 편리하게 "연합 콜드 스타트" 죽음의 함정도 해결하는 이유

§2는 말했습니다: 연합 클래스 제품의 가장 큰 죽음의 함정은 **피어 수 = 0**입니다. 우리가 선택하는 두 교두보는 **자체 피어를 가져옵니다**:

- "부모 + 아이"는 **2개의 주권 허브**에서 시작합니다;
- "가게 + 공급업체," "본사 + 프랜차이즈," "스승 + 제자"는 시작부터 **≥2개 당사자**입니다.

다시 말해, 이 종류의 사용자의 **사용 시나리오 자체가 쌍/그룹 연합**입니다 — 두 번째 피어는 영업으로 찾아야 하는 것이 아니라 사용 사례가 가져오는 것입니다. 이것은 "엔터프라이즈가 하나의 단일 배포를 구매"와 근본적으로 다릅니다: 엔터프라이즈 단일 배포는 연합 네트워크를 콜드 스타트할 수 없는 반면, 한 쌍의 가족 또는 공급망은 **자연스럽게 두 번째 노드를 가져옵니다**. 따라서 이 셀을 선택하는 것은 "가장 충족되지 않은 필요" 이면서도 연합의 네트워크 효과 콜드 스타트 문제를 "죽음의 함정"에서 "사용 사례가 가져오는 것"으로 뒤집는 방법입니다.

### 4.4 솔직한 경계 (이것은 "자동으로 이기는" 것이 아닙니다)

- **벽 A의 가격은 밀려 넘어갔지만, 벽 A의 "성숙도"는 아닙니다**: 가정/소규모 가게는 **진정으로 기본 제공** (한 줄 시작, 데스크톱 셸, 바보 증명 온보딩)을 원합니다; Gotong는 여전히 예제 우선 + 구성이 필요합니다. 가격/성능이 수요를 열었지만, **사용성이 다음 게이트입니다**.
- **신뢰/컴플라이언스는 여전히 가정과 규제받는 소규모 팀에게 어려운 게이트입니다**: 아이의 데이터를 보호하고, 법률 사무소 컨소시엄을 위해 — 감사/컴플라이언스 뒷받침 없이는 여전히 감히 사용하지 않을 것입니다.
- **배포는 여전히 비즈니스 문제이며 코드 문제가 아닙니다**: 이 셀의 사용자는 분산되어 있고 획득하기 어렵습니다; 몇 가지 더 많은 기능이 아니라 노코드 진입점 + 템플릿 갤러리 + 실제 참조 고객이 필요합니다.

## 5. 의사 결정을 위한 한 줄

> **가장 예리한 타깃은 "거버넌스/후견/교차 조직이 필요하지만 엔터프라이즈 플랫폼이 도달할 수 없고 OSS 프레임워크가 성장할 수 없는" 소규모 조직 — 가정/교육 우선, SMB 교차 조직 협업 차선.** 그들은 역사적으로 "LLM이 너무 비쌈"과 "그런 제품이 없음" 둘 다에 막혀 있었습니다; DeepSeek의 지난 두 달간의 가격 인하가 전자를 제거했고, Gotong는 정확히 후자에 대한 답이며, 이런 사용자의 사용 사례는 **자체 연합 피어를 가져와** 콜드 스타트를 편리하게 해결합니다. 경쟁자도 저렴한 DeepSeek를 사용할 수 있지만, 가격 인하가 그들 아키텍처가 부재한 교차 조직 거버넌스를 채울 수 없습니다 — **이 셀의 가격/성능 창은 구조적으로 Gotong에게 열려 있습니다.** 남은 어려운 싸움은 사용성, 신뢰 뒷받침, 배포에서 — 기술이 아닙니다.
