# 기여자 (Contributors)

<!-- doc-version: 1.0 -->
> **문서 버전 1.0** · 한국어 번역 · 최종 업데이트 2026-06-27 · 권위 있는 원본: [English](../../CONTRIBUTORS.md). 번역본이 영어 버전과 충돌하는 경우 영어 버전이 우선합니다.

이 파일은 Gotong를 만드는 사람들을 기록합니다 — 그리고 **크고 작은 모든 종류의
기여가 중요하며, 코드만이 아닙니다.** 마침내 연합이 이해되는 튜토리얼을 쓰고,
오십 명을 프로젝트로 이끈 강연을 하고, 핵심 팀이 사용하지 않는 언어로 문서를
번역하고, 신입자가 막히지 않는 커뮤니티 공간을 운영하는 것 — 그것이 *아무도 찾지
못하는 좋은 프레임워크*를 *사람들이 실제로 사용하는 좋은 프레임워크*로 바꾸는
일이며, 이곳에서 병합된 PR과 **동등한 인정**을 받습니다. **기록하기에 너무 작은
노력은 없으며, 상당한 전파 노력은 출시된 기능과 동일하게 이 기록에 등재됩니다.**

이것은 [인정 시스템 기둥
⑤](RECOGNITION-SYSTEM.md#pillar-5)의 사람 대면 절반입니다.
이 시스템의 "통화"는 프로젝트의 유일한 통화입니다: **정직한, 눈에 보이는 귀속** —
금전 없음, 토큰 없음, 포상금 없음. 그리고 이것은 **기록이지 순위가 아닙니다.** 귀하의
이름을 달아 공개적으로 *귀하가 무엇을 했는지*를 명시합니다. 청중 규모는 조작
가능하고 이를 쫓으면 프로젝트를 허영 쪽으로 끌어당기기 때문에, 숫자로 사람의 순위를
매기지 않습니다 — [우리가 하지 않는 것](RECOGNITION-SYSTEM.md)을 참조하세요.

## 기여가 기록되는 방법

[All Contributors](https://allcontributors.org) emoji 어휘를 기여 종류의 공유되고
공인된 언어로 사용합니다 — 단, 그것의 bot 또는 GitHub Action은 **사용하지 않습니다**.
이 저장소는 장부 기록에 Actions 예산을 쓰지 않으며, 수동으로 유지 관리되는 markdown
테이블이 가장 가볍고 정직하게 동작하는 것입니다. 귀하는 일반 pull request로 추가됩니다.

| Emoji | 종류 | Emoji | 종류 |
|---|---|---|---|
| 💻 | 코드 | 📹 | 동영상 |
| 📖 | 문서 | 📢 | 강연 |
| 🌍 | 번역 | ✅ | 튜토리얼 |
| 🎨 | 디자인 | 💬 | 질문 답변 / 커뮤니티 지원 |
| 👀 | Review | 📋 | 이벤트 조직 |
| 🤔 | 아이디어 & 계획 | 📝 | 블로그 글 / 작문 |
| 🐛 | 버그 리포트 | 🚇 | 인프라 |
| 🚧 | 유지 관리 | 🔌 | Adapter / 커넥터 |

네 가지 전파 종류 — 🌍 번역, 📹 동영상, 📢 강연, ✅ 튜토리얼, 📝 블로그,
💬 커뮤니티, 📋 이벤트 — 는 의도적으로 💻 코드와 **같은 테이블**에 자리합니다.
기둥 ⑤의 핵심 의미는 바로 이것입니다: 그것들은 열등한 열이 아닙니다.

## 추가되는 방법

1. **pull request를 열어** 행을 추가하세요 — 자신 또는 다른 사람의. 다른 사람의
   작업을 인정하는 것은 환영되고 권장됩니다.
2. **기준은 위 범례에서 어느 종류든 실제, 정직한 기여입니다.** 코드/문서의 경우,
   PR 자체가 증거입니다. **전파** 기여(동영상, 강연, 튜토리얼, 글, 번역, 또는
   지속적인 커뮤니티 도움)의 경우, maintainer가 볼 수 있도록 PR 설명에 **결과물
   링크를 첨부하세요**.
3. **maintainer가 병합합니다** — 작은 PR이 병합되는 것과 같은 방식으로 (lazy consensus,
   [`GOVERNANCE.md`](../../GOVERNANCE.md#how-a-change-lands) 참조). bot 없음, 의식
   없음.

기여가 **학습 자료** — Gotong를 잘 가르치는 동영상, 튜토리얼, 또는 강연 — 라면,
신입자가 안내받는 전시관인 [`LEARN.md`](LEARN.md)에도 전시될 수 있습니다. 여기에
기록되는 것과 거기에 전시되는 것은 두 가지 다른 인정 행위입니다; 훌륭한 튜토리얼은
둘 다 받을 자격이 있습니다.

## Standing — 한 행 이상

지속적인 전파 활동은 여기에 기록될 뿐만 아니라 — 프로젝트에서 실질적인 발언권을
얻습니다. `GOVERNANCE.md`의 [maintainer
사다리](../../GOVERNANCE.md#becoming-a-maintainer)는 코드와 **동등한 트랙**을
인정합니다: 여러 언어로 문서를 살아있게 유지하기, 커뮤니티 운영하기, 지속적인 교육
자료. 프레임워크 코드를 한 줄도 작성하지 않지만 매주 그 일을 하는 사람은, 사다리가
인정하려는 바로 그런 지속적인 판단력을 기여하고 있습니다.
